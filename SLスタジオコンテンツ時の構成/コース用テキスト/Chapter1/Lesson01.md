# コース概要 ～ 「一人称エンジニア」とは何か・このコースで目指す姿

## イントロダクション

いよいよ「即戦力フロントエンド開発コース（React＋Next.js＋PostgreSQL）」がスタートします！
このコースでは、これまでに学んだ知識の「点」と「点」を繋ぎ、現場で活躍できる力へと昇華させていきます。

まずは学習を始めるにあたっての前提や、目指すべきゴールについて確認していきましょう。

## 前提

本コースは、以下の4つの基礎コースを修了している（または同等の知識がある）ことを前提としています。

- **HTML・CSS初歩コース**（HTML/CSSの基礎）
- **Reactではじめるプログラミング基礎コース**（TypeScript/JSX/コンポーネント/state管理）
- **データベースコース**（PostgreSQL/pgAdmin操作）
- **Git入門コース**（CLI操作によるバージョン管理）

「ReactからAPIを叩いたことがない」「DBとフロントエンドを繋いだ経験がない」という状態から、これらを統合して一つのWebアプリケーションを作り上げるのが本コースの目的です。
もし上記のコースで未受講のものがあれば、そちらを先に完了させてから本コースに取り組んでください。

## 本コースの特徴

各基礎コースで学んだ内容は、あくまで「個別のパーツ」です。しかし実際の現場では、これらを組み合わせて開発を進める総合力が求められます。
本コースのゴールは、**既存の技術を統合し、実務レベルの「ユーザー管理システム」を一気通貫で開発する経験を積むこと**です。

具体的には、API連携、データベース操作のCRUD（作成・読み取り・更新・削除）、Gitでの差分確認、Next.js＋Storybookを用いたコンポーネント開発へとステップアップしていきます。
最終的には、 **「技術的に一人称で開発業務を遂行できる」** 状態を目指します。

※現時点で知らないワードがあっても、このコースを進めていけば理解できるようになるので安心してください！

## 一人称エンジニアとは

言葉の定義をしておきましょう。
「一人称エンジニア」とは、**「誰にも頼らず完全に独力で全てをこなす人」のことではありません**。

実務においてタスクを受け取ったら、

1. 自分で内容を分解・言語化し、
2. 提示された時間内に完了する段取りを組み、
3. 不明点があれば質問フォーマットに沿って整理し、
4. 実装を進めて、振り返りアウトプットとしてまとめる。

このサイクルを **自ら回せる人** のことです。

### 💡 現場のリアル：実は「非技術」がとても重要

ここで少し補足をしておきます。
「エンジニア」と聞くと、パソコンに向かってひたすらコードを書き続ける姿をイメージするかもしれません。
しかし、実際の現場では **「コミュニケーション能力」や「報連相」「TPOに合わせた立ち回り」といった非技術領域（ヒューマンスキル）が非常に重要** です。

「コードは完璧だけど進捗の報告が全くない」「わからないのに何日も一人で抱え込む」といった状態では、チームとして仕事を進めることができません。

一方で、こうした「空気を読む力」や「対人コミュニケーション」の機微をテキストの学習だけで身につけるのは非常に難しいのが実情です。
そのため、本コースでは **ヒューマンスキル全般（非技術領域）についてはスコープ外** としています。

その代わり、タスク分解や質問フォーマットの作成といった **「フォーマットに沿って情報を整理する技術」** にフォーカスして学習を進めます。
対人コミュニケーションやビジネスマナーについては、 **現場デビューしてから先輩やチームの姿を見て、実践の中で身につけていきましょう。**

## 学習の準備

本コースを進めるにあたり、以下の環境や操作が問題なく行えるかチェックしてみましょう。
これらはこれまでのコースで学習済みの内容が中心です。

- [ ] **エディタの準備**: VSCodeがインストールされており、拡張機能の `GitLens` を導入できる準備があること
  - 拡張機能の `GitLens` は今後の学習で導入をするので今はVSCodeがインストールされていればOKです。
- [ ] **Node.jsの確認**: `node -v` コマンドでバージョンが表示されること（v18以上推奨）
- [ ] **プロジェクト作成**: `npm create vite@latest` コマンドを実行し、プロジェクトのひな型が作成できること
- [ ] **データベース環境**: ローカルPCで `pgAdmin` が起動し、PostgreSQLデータベースに接続できること
- [ ] **Gitの基本**: `git commit` などの基本的なコマンドの意味を理解していること

もし「やり方を忘れてしまった」という項目があれば、過去のコースのテキストをサッと見直しておくと、今後の学習がスムーズになります。

## このレッスンのまとめ

- **コースの目標**: React・Next.js・PostgreSQL・Gitを統合して一気通貫でWebアプリを作り上げること。
- **一人称エンジニアとは**: 1人で全てこなすのではなく、「タスク分解・段取り整備・質問・実装・振り返り」のサイクルを自ら回せる人のこと。
- **現場のリアル**: 実務では「コミュニケーション」などのヒューマンスキルが非常に重要。本コースではその基礎となる「フォーマットに沿って情報を整理する技術」に特化して学ぶ。
- **学習の準備**: 過去のコースで学んだ各種ツール（Node.js, Vite, pgAdmin, Git）の基本操作ができているか確認しておく。

準備が整ったら、次のLessonへ進んでタスクをこなすための「技術的な基盤」について学んでいきましょう！
