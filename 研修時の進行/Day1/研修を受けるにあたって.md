# 研修を受けるにあたって

## 共有を受けた評価基準を意識して日々取り組む

## 各種フォーマット

### 指示を言語化する

```markdown
【概要】
・対象となるタスク名などを記載する

【期限はいつまでか】
・提出の期限を記載する

【タスクの作業時間】
・タスクごとの想定時間を記載する

【タスクの詳細】
・作業内容を記載する

【確認事項】
・事前に確認や質問があれば記載する
```

◆参考例◆

```markdown
【概要】
ユーザー管理システムの追加機能開発
ユーザーが一覧で表示される画面や新規登録画面を作成し機能を完成させる

【期限はいつまでか】
2024年12月〇日14:00までの完了

【タスクの作業時間】
作業時間：60分
12月〇日 11:00 完了予定

【タスクの詳細】
・環境構築とプロジェクト準備
・ユーザーデータの一覧機能を実装する
　・表示用のコンポーネント作成
　　・対象ファイル：components/UserList.tsx
　　・ユーザーデータの取得しstate管理（fetch利用）
　　・ユーザーデータを表示する処理（map利用）
　・動作確認
・ユーザーデータの新規登録機能を実装する
　・新規登録用のコンポーネント作成
　　・対象ファイル：components/RegisterForm.tsx
　　・入力フォーム作成
　　・入力値を利用できるようstate管理
　　・state変数を利用して登録する処理
　　　※可能なら一覧画面に遷移させる
　・レイアウト調整
　・動作確認
　　⇒一覧上に表示されるかも合わせて確認する

【確認事項】
・データ取得方法について
・データ登録方法について
　　登録の処理はどのように行う必要があるのかを確認したい
```

確認事項がある場合は必ず講師に確認を行ってください。
上記の例では最低限の記載としていますが、より詳細に記載しても問題ありません。
以下の質問フォーマットを流用して確認や質問を行いましょう。

### 質問 / 報告 / 相談 する

```markdown
【概要】
・質問、報告、相談のどれか
・対象がなにか

【現在の状況】
・現在の状況がどうなっているのか

【調査内容 / 詳細】
・調べた原因
・報告、相談内容
など
```

質問の場合は【調査内容】
報告や相談の場合は【詳細】として記載してください。

◆参考例（質問）◆

```markdown
【概要】
ユーザー管理システムのタスク2について不明点があるため質問させていただきたいです。

【現在の状況】
資料を確認し、データを登録する必要があることは把握できているのですが、
具体的にどのように実装すれば良いのかが理解しきれていない状況となります。

【調査内容】
・調査してわかっていること
AIにて実装手法を確認し、fetchを利用することで実現可能である。
公式のリファレンスにて文法を確認し、SLスタジオで学習したコードと類似していることを確認。

・不明点
調査して出てきたサンプルコードの記載方法がこれまでに学習した記載方法を異なるため、
どの程度参考にしてもよいのかがわかっていない。
```

### タスクスケジュールの相談や報告

```tsx
【対象となるタスク】

・
```

### 口頭での注意点

質問や報告などを行う際は上記のフォーマットをベースとして口頭で行ってもらいます。
その際に、以下の点に注意してください。

- いきなり内容を話し始めない
    - 必ず「○○さん、今お時間よろしいでしょうか？」などの確認を行う
    - 相手の状況によってはすぐに話を聞けない場合もあります
- フォーマットを利用する
    - 「ここわからないんですけど教えてください」と話し始めると
    共通認識がない状態では何タスクの何を知りたいのかが全く分かりません
    - 質問なのか報告なのか相談なのか…前提を共有して確認しましょう
- スムーズなコミュニケーションを心がける
    - 事前に内容をまとめてから会話するようにしましょう
    - レスポンスはしっかりと行う
    - わからない場合は再度質問する
- 自分のターンで終わらせる
    - 回答をもらい終話する際は必ず「承知しました」「ありがとうございました」のように
    自分から終わるようにしてください
    - 指示や回答に無反応で終わらないでください

### Slackを活用しよう

講師が離席中の場合や、他のメンバーが質問や報告をしているタイミングでは、
Slackで連絡を投げるようにしてください。
上記のタイミング以外に事前に文章で送ってから声をかける等の使い方も〇です。

いつでも口頭で話せる状況とは限らないため、文章でも投げられるようにしておいてください。

## エラーや問題が起きた場合

エラーが発生した場合は、まずはエラーの内容を読んでみましょう。
英語がわからなくても翻訳を活用すれば意味合いは理解できるはずです。

そのあとは以下の流れを行って解決に向けて進めてください。

1. エラーを読む（翻訳してもOK）
2. エラー内容を検索する
3. 該当箇所を探す
4. 対応策を調査する
5. 試してみる

### エラー内容を調査しよう

- Googleでエラーの原因例を探ってみる
- AIに状況を伝えて解決策を聞いてみる

エラーの内容がわかったら、次は調査を行いましょう。
ネットの海は広いですが、全く同じ状況があるとは限りません。
いくつかの情報を確認しつつ、自分の状況に当てはめて整理してみましょう。

エラーに限らず、「画面が真っ白」「起動しない」等も同じように調査して進めてください。

### 時間の配分に気を付ける

自己学習とは異なり、仕事には期限が設けられています。
自身で調べることも重要ですが、調査に時間を掛けることができる時間は限られています。

タスクの期限にもよりますが、**どんなに長くても10分調べてわからなければ絶対に質問してください**。場合によっては3~5分以内に質問する必要もあるため、時間は常に意識しましょう。

## 報連相を適切に行う

報告、連絡、相談、これらは密に行ってください。
質問についても同様です。

以下をルールとしての参考にしてください。

- スケジュール関係
    - 見積もりの共有と相談
    - タスクごとのタイミングでの完了報告
    - 順調な場合の定期報告
    - 進捗遅れの相談と報告
        - 期限から逆算して残り時間半分あたりを目安
- タスク関連
    - やるべきことの方向性に相違がないか確認
        - 実装内容に問題がないかも事前に確認する
    - タスクの不明点や確認事項があれば質問する
        - 気付いた時点ですぐに声をかける
        - わからない状態では絶対に作業しない
    - エラーなどで詰まった際の質問
        - 3分~5分を目安に調べ、それでもわからない場合は質問する
        - 調査内容に問題がないかも確認のため質問する
