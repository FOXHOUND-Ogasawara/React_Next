🛠️ Chrome DevTools 実践学習ページ

🎯 ページ全体の目標

Web開発において必須のツールであるChrome DevToolsの基本操作を習得します。特に「Elements」「Console」「Sources」を重点的に学び、他のパネルについても知っておくと便利なレベルで体験します。

📌 導入：DevToolsの開き方とパネルの概要

✅ 実践タスク： まず、DevToolsを開いてみましょう！
  • Windows: F12 または Ctrl + Shift + I
  • Mac: Cmd + Option + I
  • または： 任意の要素を右クリック → 「検証」を選択

DevToolsの主要パネル

  • Elements: HTML/CSSの構造確認・編集
  • Console: JavaScript/TypeScriptのログ出力・エラー確認
  • Sources: ブレークポイントを使ったデバッグ
  • Network: 通信状況の確認
  • Performance: 処理速度の記録と可視化
  • Memory: メモリ使用量の追跡
  • Application: ストレージやCookieの確認

🐛 セクション 4：Sources（ブレークポイントとデバッグ）

✅ 実践タスク：
  1. DevToolsの「Sources」タブを開く
  2. 左ペインでこのHTMLファイルを探してクリック
  3. 下の debugIncrement() 関数の中の行番号（例：const newVal = prev + 1;）をクリックしてブレークポイントを設置
  4. 「デバッグ実行」ボタンをクリック
  5. 処理が停止したら、右側の「Scope」で変数 prev と newVal の値を確認
  6. 「Step over」ボタン（右上の ⤵ アイコン）で1行ずつ進める
  7. 「Resume」ボタン（▶ アイコン）で処理を続行
// デバッグ対象の関数
function debugIncrement() {
  const prev = debugCounter;
  const newVal = prev + 1;  // ← ここにブレークポイントを設置
  console.log('[Debug]', '+1 →', newVal);
  debugCounter = newVal;
  document.getElementById('debug-counter').textContent = debugCounter;
}
0

💡 デバッグツールの使い方

  • Resume (▶): 処理を続行（次のブレークポイントまで）
  • Step over (⤵): 次の行に進む（関数内に入らない）
  • Step into (↓): 関数の中に入る
  • Step out (↑): 現在の関数から抜ける

📚 セクション 5：その他の便利なパネル

5.1 Network（通信状況の確認）

確認ポイント

項目 説明
フィルタ 「Fetch/XHR」をクリックして、API通信だけを絞り込み表示
Status 200（成功）、404（URL間違い）、500（サーバーエラー）などを確認
Headers リクエストURLやメソッド、パラメータを確認
Response APIから返ってきたJSONなどのデータを確認

5.2 Performance（処理速度の記録と可視化）

使い方

  • 記録開始: 「Record（●）」ボタンで計測開始
  • 記録停止: 操作後に「Stop」で計測終了
  • CPUグラフ: 処理が重い部分（山の大きい部分）を特定
  • Frames: フレーム描画の安定性（緑はOK、黄・赤は注意）
  • Summary: 処理時間の内訳を確認（黄：Scripting、緑/青：描画）

5.3 Memory（メモリ使用量の追跡）

確認ポイント

  • Heap snapshot: その瞬間のメモリ使用量を撮影
  • Shallow Size: オブジェクト自体のメモリ量
  • Retained Size: そのオブジェクトが保持している総メモリ量（大きい場合は要注意）

5.4 Application（データ保存場所の確認）

確認ポイント

  • Local Storage: ブラウザに保存された一時データ
  • Session Storage: タブを閉じると消えるデータ
  • Cookies: ログイン状態などを維持するデータ
