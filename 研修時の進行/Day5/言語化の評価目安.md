# 言語化の評価目安

## 前提として

いくつかの類似した質問や確認事項が想定されます。

- コンポーネントの確認方法
- Rechartsの使い方

これらは誰が担当するかによって全員から質問や確認される可能性があります。

その場合は最初の確認時に補足として全体に共有してあげてください。

### 表示の確認について

基本的には `Dashboard.tsx` にてコンポーネントを呼び出す方針で問題ありません。

そのため、「確認しやすいように `Dashboard.tsx` に追記して確認できればOK」と伝えてください。

注意点として、**複数人が `Dashboard.tsx` を触り表示を各々で表示確認を行う場合、確実にコンフリクトが発生します**。

これについては明言せず、コンフリクトを解消するのも課題の一つとして見守りましょう。

また、メンバーから「コンフリクトを防ぎたい」という趣旨の相談があった場合は「コミットする前に `Dashboard.tsx` を元の状態に戻せば問題ない」ことを伝えてあげてください。

この時、**動作確認は絶対に行うこと**をルールとし、最後の提出時に動作不良や仕様漏れがあった場合は指摘対象となることも伝えましょう。

### Rechartsの使い方について

公式ドキュメントの確認方法を画面を見ながら共有してあげてください。

上部メニューの「API」を開くとサイドメニューに各種グラフコンポーネントのサンプルや使い方などが閲覧できます。

ドキュメントを読むことも仕事の一環なので、基本的にはそこまでのアドバイスで問題ありません。

個別の質問として具体的な利用方法を聞かれた場合はフォーマットに則り、どこまで調べたのかを確認しつつ回答するようにしてください。

### ユーザー統計情報のグラフ作成

- 観点
    - フォーマットを利用しているか
    - タスクの見積もり時間ができているか
    - 棒グラフの表示に関する記載があるか
    - 事前にわかる範囲で確認事項を記載しているか
- 言語化例

```
【期限はいつまでか】
202x/xx/xx xx:xxまで

【タスクの作業時間】

　・userData.ts内のデータを利用する
　　対象データ：month, newUsers, activeUsers

　タスク：「ダッシュボードの修正」と内容が被るためどのように反映するかを確認したいです
・Rechartsの棒グラフに関して
　ドキュメントを見つつ作業予定ですが、一度利用方法について確認したいです
```
