# ダッシュボード画面を開発する

## 準備

リモートリポジトリをクローンし、環境構築を行いなさい。

対象となるリポジトリは、Slackにて共有します。

`npm run dev` を実行して、画面が表示できることを確認すること。

### 作業内容について

以下のタスクを確認し、作業を分担しつつ実装を進めること。

また、作業者の中から1名をリーダーとし、プルリクエストの管理を行うこと。

＜タスク＞

- ユーザー統計情報のグラフ作成
- 売上データの折れ線グラフ作成
- ダッシュボードの修正
- 商品一覧ページの追加
- 商品カテゴリ別の割合グラフ作成
- 商品別の売上グラフ作成

＜追加タスク＞

- 収益情報の折れ線グラフ作成
- カテゴリー売上のグラフ作成
- 期間ごとの売上データの折れ線グラフ作成

データについては、 `src/data` 内に格納してあるものを利用すること。

グラフの作成は、Rechartsの公式ドキュメントを参考にすること。

URL：[https://recharts.org/en-US/](https://recharts.org/en-US/)

レイアウトについては、MUIの公式ドキュメントを参考にすること。

URL：[https://mui.com/](https://mui.com/)

## タスク詳細

### ユーザー統計情報のグラフ作成

ダッシュボード上に表示するユーザー統計情報の棒グラフを作成する。

作業内容

- `src/components/` ディレクトリに `UserStatsChart.tsx` ファイルを作成。
- `Recharts` ライブラリを使用して棒グラフを実装。
- モックデータ（`userData.ts`）を利用し、データをグラフに表示。
    - 例：月、増加数、アクティブ数などの統計

### 売上データの折れ線グラフ作成

ダッシュボード上に表示する折れ線グラフを作成する。

作業内容

- `src/components/` ディレクトリに `SalesChart.tsx` ファイルを作成。
- `Recharts` ライブラリを使用して折れ線グラフを実装。
- モックデータ（`salesData.ts`）を利用し、データをグラフに表示。
    - 例：日、売上合計 + 週、売上合計 + 月、売上合計のデータ
- 日別、週別、月別のデータ切り替え機能を実装。

### ダッシュボードの修正

実装した `UserStatsChart` と `SalesChart` などのグラフをダッシュボードに表示する。

作業内容

- `Dashboard.tsx` にて、サンプルコンポーネント `PlaceholderChart` を削除。
- 代わりに、実装した `UserStatsChart` と `SalesChart` をインポートして表示。
- 追加で `CategoryPieChart.tsx` と `SalesPieChart.tsx` をインポートして表示。

### 商品一覧ページの追加

商品の一覧をテーブル表示するページを作成する。

作業内容

- `Products.tsx`を`src/pages/`に作成。
- 商品のモックデータを作成。
    - ファイル名： `productData.ts`
    
    ```tsx
    // data/productData.ts
    export interface Product {
      id: number;
      name: string;
      category: string;
      price: number;
    }
    
    export const products: Product[] = [
      // 6種類の商品データを追加
    ];
    ```
    
    - データ名を `products` として6種類以上の商品データを追加。
- `App.tsx`と`SideMenu.tsx`に商品一覧ページへのルートとメニュー項目を追加。
    - URLについては確認すること。
- 商品ID、商品名、カテゴリー、価格をテーブルで表示。

### 商品カテゴリ別割合の円グラフ作成

ダッシュボード上に表示する円グラフを作成する。

作業内容

- `src/components/` ディレクトリに `CategoryPieChart.tsx` ファイルを作成。
- `Recharts` ライブラリを使用して円グラフを実装。
- モックデータ（`productData.ts`）にデータを追加。
    
    ```tsx
    // data/productData.ts
    export interface Category {
      id: number;
      category: string;
      value: number;
    }
    
    export const categoryData: Category[] = [
    	// 6つのデータを追加
    ];
    ```
    
    - データ名を `categoryData` として6つ以上のカテゴリデータを追加。
- モックデータを利用してデータをグラフに表示。

### 商品別の売上金額の円グラフ作成

ダッシュボード上に表示する円グラフを作成する。

作業内容

- `src/components/` ディレクトリに `SalesPieChart.tsx` ファイルを作成。
- `Recharts` ライブラリを使用して円グラフを実装。
- モックデータ（`salesData.ts`）を利用し、データをグラフに表示。

## 各種機能詳細

### ページ構成

- ダッシュボードページ
    - URL： `/`
    - サイドメニューとヘッダーが表示され、メインエリアにユーザー統合グラフと売上グラフを配置する。
    - テーマ切り替えスイッチがヘッダーに設置されている。
- 通知センターページ
    - URL： `/notifications`
    - 通知一覧が表示され、通知をクリックすると詳細が表示される。

### 動作イメージ

- ナビゲーション（ルーティング）
    - サイドメニューから各ページに移動可能
    - 各ページはReact Routerでルーティングされている
- テーマ切り替え
    - ダークモードとライトモードを切り替えられる
    - 選択したテーマはアプリ全体に適用される
- グラフ表示
    - ユーザー統計と売上データがグラフで視覚的に表示される
    - 売上データは期間（日別、週別、月別）を切り替えて表示可能
- 通知センター
    - 通知をクリックすると詳細が表示される

## 作業の流れ

### はじめに

プロジェクトをクローンして、環境構築を行いましょう。

その後、メンバーから一人をリーダーとして決定してください。

リーダーには、次の作業を担ってもらいます。

- この後の作業分担
- 各作業内容のマージ
- 完了報告
    
    

誰がどのタスクを着手するのかを決め、作業を進めていきましょう。

### Gitのルール

該当のプロジェクトをクローン後、リーダーが以下のブランチを作成する。

```bash
team-[チーム名]-[入社日]
```

- チーム名はリーダーの苗字から頭文字のアルファベットを利用
- 入社日は `1001` や `1015` のように数字四桁を繋げた形式とする
- 例： `team-y-1001`

ブランチ作成後にリモートリポジトリへプッシュし、他メンバーも `fetch` → `merge` を行いローカルリポジトリへ反映する。

このリポジトリに対して作業を集約する形として運用を行うこととする。

### 作業を開始する

チーム用のブランチから、自身の作業用ブランチを作成します。

`team-[チーム名]-[入社日]_iwai` のように、チーム名の後ろに `_` を繋げて自身の名前を付けましょう。

ブランチを作成したら、リモートリポジトリに反映し、割り振られた機能開発を進めてください。

### 機能を統合する

自分が担当する機能が完成したら、GitHubを利用して自身のチーム用ブランチにマージリクエストを行いましょう。

その後、リーダーに共有し、マージを行ってください。

すべての機能がマージされ、問題なく動作すればタスク完了です。
