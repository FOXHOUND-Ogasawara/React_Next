## 本プロジェクトについて

### お題

本プロジェクトはユーザー管理システムを模したアプリケーションとなります。

ここまでの研修で学んだAPI通信やCRUD処理を利用し、コンポーネントを実装していくことになりますので、身に着けた知識を活かしつつ完成に向けて頑張ってください。

実際に着手してもらうにあたって、以下の部分を意識し作業を進めてください。

- あくまでもReactを用いたプロジェクトである
    - 文法やルールが一部異なりますが、あくまでもReactです。
    - 前提の知識や技術はすでに学習しているので安心してください。
- これまでとは異なる実装方法が含まれている
    - Next.jsやStorybookという新たな技術を利用することになります。
    - 確認や質問を怠るとそれだけで進捗に影響が出ることになります。
- 各タスクを完了したらリモートリポジトリにプッシュをするようにしてください。

### 大きく異なるポイント

- ルーティング方法
    - Nextのappルーターという技術を利用しています。
    - `app` 配下に `page.tsx` を作成することで自動的にルーティングされる仕組みです。
- APIを利用した通信処理
    - `fetch` を使わずに簡潔な記述で通信処理が可能です。
    - ファイル内のどこかに参考になる記述があるため活用してください。

### Storybookとは

`components` 配下に `コンポーネント名.stories.tsx` というファイルが存在しています。

これまで、コンポーネントを作成しても別のコンポーネントから呼び出さない限りは表示の確認はできませんでしたが、Storybookを利用することでコンポーネント単体での確認やテストが可能となり、さらにはコンポーネントの使い方をドキュメント化することができる便利な仕組みです。

最初は難しく感じるかもしれませんが、記述方法にはフォーマットがあり、基本的にはサンプルをまねて作ることで実装することが可能です。

Day1時点では「コンポーネントを単体で確認できる機能」くらいの認識で捉えてもらって大丈夫なので、まずはStorybookの作り方やルールを覚えてください。

## Gitのルール

プロジェクトのクローン後、タスクごとにブランチを切ること

```bash
main
	┗develop
		┗Day1_task1-1
			┗Day1_task1-2
				┗Day1_task1-3
					・・・
```

- まず、developブランチを切る
- その後、`DayX_タスク`というブランチを切る（例：Day1_task1-1）
- 動作確認後、問題なければpushをしてdevelopブランチにPRを出すこと
- PRの承認は任意とします
- 明日以降実施するDay2,3についても`Day2_task1-1`のようにブランチを切ること

こうすることで、mainブランチは汚れないので自己学習で再度1から実装したいということがあってもmainに切り替えればクローン直後の状態になります。

## 新規登録機能関連のタスク

ユーザーの新規登録用フォームコンポーネントを作成します。

下記のソースコードを対象ファイルにコピーして作業を行ってください。

- 対象となるファイル
    - `components/RegisterForm.tsx`

- 仕様
    - コンポーネント名が `RegisterForm` である
    - コンポーネントは `RegisterFormProps` で定義した内容を受け取ることができる

        - 「登録」ボタンを押すことで登録が可能
        - 登録成功後はユーザー一覧画面に遷移される
            - 遷移する仕組みについてはヒントを参考にする

必要ファイルはプロジェクト内の「必要ファイル」フォルダにあります。

<aside>
💡

ヒント

データ登録成功時のページ遷移を関数としてPropsから受け取る方法。
利用するPropsは以下を参考にしてみましょう。

```tsx
interface RegisterFormProps {
  onSuccess?: () => void;
  onError?: (error: any) => void;
  disabled?: boolean;
}
```

</aside>
