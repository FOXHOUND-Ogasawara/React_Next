# Git_台本

前回のデータベース基礎に引き続いて、今回はGitとGitHubについて、その基礎となる部分をまとめてみましょう。ソースの内容に基づき、Git/GitHubの基本的な概念や使い方を分かりやすく解説します。

### GitとGitHubとは

Gitは、**ファイルの変更履歴を記録・管理するための分散型バージョン管理システム**です。これにより、いつ、誰が、どのような変更を加えたかを追跡したり、過去の状態に戻したりすることが容易になります。

GitHubは、**Gitを使って管理しているコードをインターネット上で共有するためのサービス**です。GitHubを利用することで、**リモートリポジトリ**と呼ばれるインターネット上の保管場所にコードを置き、複数人で共同で開発を進めることができるようになります。

バージョン管理をすることで、ファイルやフォルダなどを履歴ごとに保存して管理し、任意のタイミングで好きなように切り替えることができます。

### Gitのインストールと準備

まず、お手元のコンピュータにGitをインストールする必要があります。

**Git for Windowsの公式ページ**からインストーラーをダウンロードし、基本的にデフォルト設定でインストールを進める手順が示されています。

インストール後、Git Bashなどのターミナルで `$ git --version` コマンドを実行することで、

正しくインストールされているか確認できます。

Git Bashはターミナルの一種で、コマンドプロンプトのようなものです。

次に、GitHubを利用するためにアカウントを作成します。**GitHubの公式ページ**にアクセスし、「Sign up」から必要な情報を入力してアカウントを作成します。

セキュリティ向上のため、**2要素認証を設定すること**が推奨されています。

スマートフォンのGoogle Authenticatorアプリなどを使用し、QRコードを読み取って

設定を行います。リカバリーコードは、端末買い替え時などに必要となるため、大切に保管しておきましょう。

### Gitの基本的なコマンドと概念

Gitを使ったバージョン管理は、主にGit Bashなどのターミナルでコマンドを実行して行います。

コマンド入力の際は、**先頭に `git` と入力し、半角スペースを一つ空けてコマンド名を入力します**。

コマンドの前に表示される `$` は、コマンド入力欄であることを示す記号であり、実際には入力しません。

以下に、ソースで紹介されている基本的なコマンドとその役割を挙げます。

- `$ git init`: 現在のディレクトリでGitによる管理を開始します。
これにより `.git` フォルダが追加されます。
- `$ git config --global user.email 自分のメールアドレス`: Gitを使うユーザーのメールアドレスを設定します。
- `$ git config --global user.name 自分の名前`: Gitを使うユーザーの名前を設定します。
名前は「名前-苗字」、メールアドレスは会社から割り振られたものを使用することが推奨されています。設定内容は、メールアドレスや名前の部分を省略してコマンドを実行することで確認できます。
- `$ git add -A`: **変更点をステージングに追加**します。
次に説明する `commit` とセットで使うことが多いです。
- `$ git commit -m "コメント"`: **ステージングに追加された変更点を履歴として保存**します。
この操作を「コミットする」と呼びます。
コミット時には、どのような変更を行ったか分かりやすいようにコメントを付けます。
- `$ git status`: **ファイルなどに変更が加えられた状態を確認**できます。
変更があったファイル名などが表示されます。
- `$ git log`: **コミット履歴を表示**します。過去のコミット内容を確認する際に使用します。
- `$ git branch`: **作業場（ブランチ）の一覧を表示したり、新しいブランチを作成**したりします。タスクごとにブランチを作成して作業を進めることが多いです。
ブランチを作成することを「ブランチを切る」と呼びます。
- `$ git checkout ブランチ名`: **作業場（ブランチ）を切り替えます**。
特定のブランチでの作業を始めたい場合や、過去の履歴（ブランチ）の状態に戻したい場合に使用します。
これにより、ファイルの内容がそのブランチの状態に瞬時に変化する、というバージョン管理の利点を体験できます。

### GitHubとの連携（ローカルとリモート）

ローカルコンピュータ上にあるGit管理されたプロジェクトを**ローカルリポジトリ**と呼ぶのに

対し、GitHubなどのインターネット上に存在するものを**リモートリポジトリ**と呼びます。

共同作業を行うには、ローカルリポジトリとリモートリポジトリを連携させる必要があります。

連携の手順は以下のようになります。

1. **GitHub上にリモートリポジトリを作成**します。GitHubにログインし、
「Create repository」などをクリックしてリポジトリ名（例: git-sample）などを
設定して作成します。
2. **ローカルリポジトリとリモートリポジトリを紐付け**ます。
ローカルプロジェクトのディレクトリで `$ git remote add origin リモートリポジトリのURL` コマンドを実行します。
`origin` はリモートリポジトリの慣習的な名前です。
3. **ローカルでの変更内容をリモートリポジトリに反映**します（**プッシュ**）。
初めてプッシュする場合や新しいブランチをプッシュする場合は、
`$ git push -u origin ブランチ名` コマンドを使用します。
一度紐付けた後は `$ git push` でプッシュできます。
4. **リモートリポジトリの変更内容をローカルリポジトリに取り込み**ます（**プル**）。
これには、まず `$ git fetch` でリモートの差分情報を取得し、次に `$ git merge` で
ローカルに反映するという手順があります。GitHub Desktopでは、`pull` コマンドが `fetch` と `merge` を合わせた操作となります。

### チーム開発の流れとコンフリクト

Gitを使ったチームでの開発は、一般的に以下のような流れで進められます。

1. **元となるブランチ（例: masterやdevelop）から、自分の作業用のブランチを作成**します。
2. **作業用ブランチでタスクに応じた作業を行います**。
3. **作業の区切りごとにコミットして履歴を保存**します。
4. **作業が完了したら、自分のブランチをリモートリポジトリにプッシュ**します。
5. **GitHub上で**、自分の作業用ブランチの変更内容を、統合したいブランチ（例: masterやdevelop）に取り込んでもらうための**プルリクエストを作成**します。
6. **他の作業者（リーダーなど）がプルリクエストの内容を確認**し、問題がなければ変更内容を統合したいブランチに**マージ**します。
- *コンフリクト（競合）は、**同じファイル内で複数の人が異なる変更を行った場合に発生**し、自動的に変更を統合できない状態です。
特に複数人で同じファイルを同時に編集して作業用ブランチを作成し、developブランチに対してプルリクエストを投げると、コンフリクトが発生することがあります。

コンフリクトが発生した場合、手動で該当箇所を修正する必要があります。ソースでは以下の手順が示されています。

1. ベースとなるブランチ（例: develop）を最新の状態にします (`$ git checkout develop` -> `$ git fetch` -> `$ git merge`)。
2. 自分の作業ブランチに戻ります (`$ git checkout develop_YOURNAME`)。
3. 最新状態のベースブランチの内容を作業ブランチに取り込みます (`$ git merge develop`)。
4. この操作によりVS Codeなどでコンフリクト箇所が表示されるので、手動で修正します。
5. 修正が完了したら、再度コミットしてプッシュし、
GitHub上のコンフリクト表示が解消されたことを確認します。

### GitHub Desktop

ソースでは、Gitのコマンド操作だけでなく、**GitHub DesktopというGUIツールを使った操作方法**も紹介されています。
GitHub Desktopを使うと、リポジトリの作成やクローン、変更内容の確認、コミット、プッシュ、ブランチの切り替え、プルリクエストの作成といった一連の操作を、
画面上で行うことができます。

コマンド操作では変更内容を `$ git status` や `$ git log` で確認しますが、GitHub Desktopでは**画面上で修正前後の差分を分かりやすく確認できます**。不要なコメントや動作確認用のコードが残っていないかなど、**コミット前に必ず内容をチェックする習慣をつけること**が重要です。

### まとめ

GitとGitHubは、ソフトウェア開発において非常に重要なツールです。
Gitによるバージョン管理と、GitHubを使ったチームでのコード共有・共同作業のプロセスを
理解することで、効率的かつ安全に開発を進めることができるようになります。
資料で紹介された基本的なコマンドとGitHubでの操作方法を習得し、実際に手を動かして慣れていくことが習得への近道となるでしょう。
