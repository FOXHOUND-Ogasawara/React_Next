# ユーザー管理サイト_Day3_台本

Day3では、アプリケーションのUI/UXを向上させるために、いくつかの共通コンポーネントを作成し、既存機能に統合していく作業が中心となります。

主なタスクは以下の通りです:

1. **ボタンコンポーネントの実装**
2. **カードコンポーネントの実装**
3. **モーダルコンポーネントの実装**
4. **ストーリーの追加実装**
5. **アプリケーションへの統合**

これらのタスク1〜3は、コンポーネントの作成とStorybookの作成がセットになっています。

各タスクにはTODOが記載されており、そのままだと読み解くのが難しい場合があるため、

コンポーネント作成とStorybook作成の内容を合わせて確認し、ストーリーブック側に記載されているヒントも参考に進めることが推奨されています。

以下に各タスクの詳細を記載します。

### タスク1：ボタンコンポーネント

多様なスタイルや機能を持つカスタムボタンコンポーネントを作成し、Storybookを作成し、既存の削除ボタンを差し替えます。

1. **タスク1-1：ボタンコンポーネントの作成**
    - 対象ファイルは `components/parts/CustomButton.tsx` です。
    - MUIを利用してレイアウトを整えます。
    - `CustomButtonProps` インターフェースは `ButtonProps` を拡張し、`variantType` ('primary' | 'secondary' | 'danger') を持ちます。
    - 提供されたコードスニペットのTODO箇所（`variantType` に応じて `color` を変化させる部分と、`<Button>` タグの実装部分）を修正します。`color` に設定する色は調べて実装する必要があります。プロップスには `color`、`variant` を設定し、最後に `{...props}` を設定します。
2. **タスク1-2：ストーリーブック作成**
    - 対象ファイルは `components/parts/CustomButton.stories.tsx` です。
    - `CustomButton` コンポーネントのストーリーを作成します。
    - 提供されたコードスニペットのTODO箇所（メタデータのエクスポート、ストーリーの定義）を修正します。
    - デフォルトの `Primary` ストーリーを参考に、`Secondary` と `Danger` のストーリーを設定します。
3. **タスク1-3：DeletedUserButtonの改修**
    - 対象ファイルは `components/UserCard.tsx` です。
    - Day2で実装した `UserCard` コンポーネント内の削除ボタンを、作成した `CustomButton` に差し替えます。
    - 差し替え後も、ボタンの仕様はDay2で実装した `DeletedUserButton` と同じ挙動を維持する必要があります。

### タスク2：カードコンポーネント

情報を表示するための再利用可能なカードコンポーネントを作成し、Storybookを作成し、`UserCard` コンポーネントを差し替えます。

1. **タスク2-1：カードコンポーネントの作成**
    - 対象ファイルは `components/parts/CustomCard.tsx` です。
    - MUIを利用してレイアウトを整えます。
    - `CustomCardProps` インターフェースは `title: string; description: string; actions?: React.ReactNode;` を持ちます。提供されたコードスニペットのTODO箇所（インターフェースの修正）を修正します。
    - `<CardContent>` 内のTODO箇所（`title` と `description` を表示）を実装します。
2. **タスク2-2：ストーリーブック作成**
    - 対象ファイルは `components/parts/CustomCard.stories.tsx` です。
    - `CustomCard` コンポーネントのストーリーを作成します。
    - 提供されたコードスニペットのTODO箇所（メタデータ、ストーリーの定義）を修正します。
    - `Default` ストーリーでは、`title`、`description`、`actions` にそれぞれ値を設定します。`actions` には `CustomButton` を含める例が示されています。
    - `WithoutActions` ストーリーでは、`title` と `description` のみを設定し、アクションがない場合の例を示します。
3. **タスク2-3：UserCardの改修**
    - 対象ファイルは `components/UserList.tsx` です。
    - `UserList` コンポーネント内で使用されている `UserCard` コンポーネントを、作成した `CustomCard` に差し替えます。
    - 差し替え後も、仕様は元の `UserCard` と同じになるようにします。

### タスク3：モーダルコンポーネント

ダイアログやモーダルを表示するための再利用可能なコンポーネントを作成し、Storybookを作成し、削除時の確認モーダルを差し替えます。

1. **タスク3-1：モーダルコンポーネントの作成**
    - 対象ファイルは `components/parts/CustomModal.tsx` です。
    - MUIを利用してレイアウトを整えます。
    - `CustomModalProps` インターフェースは `open: boolean; title: string; content: string; onClose: () => void; onConfirm?: () => void;` を定義します。提供されたコードスニペットのTODO箇所（propの設定）を修正します。
    - コンポーネントは `open`、`title`、`content`、`onClose`、`onConfirm` を受け取れるようにします。
    - `<Box>` のスタイルとして、画面中央に配置する設定が提供されています。
    - モーダル内にはタイトル (`Typography` variant="h6") とコンテンツ (`Typography` sx={{ mt: 2 }})、そして「キャンセル」ボタンと（`onConfirm` がある場合の）「確認」ボタンが含まれます。
2. **タスク3-2：ストーリーブック作成**
    - 対象ファイルは `components/parts/CustomModal.stories.tsx` です。
    - `CustomModal` コンポーネントのストーリーを作成します。
    - 提供されたコードスニペットのTODO箇所（メタデータ、ストーリーの定義、デフォルトストーリーの作成）を修正します。
    - `Default` ストーリーでは、`useState` を使用してモーダルの開閉状態を管理し、ボタンをクリックすることでモーダルが開閉するように実装します。
    - `CustomModal` コンポーネントにProp（`open`、`onClose`、`onConfirm`）を渡します。`onClose` は `setOpen(false)` を渡し、`onConfirm` は `alert()` でクリックを知らせた後、`setOpen(false)` を渡すように指示されています。
3. **タスク3-3：DeletedUserButtonの改修**
    - 対象ファイルは `components/parts/CustomButton.tsx` です。
    - Day2で実装した `DeletedUserButton` で使用していた `confirm` 関数による確認画面を、作成した `CustomModal` に差し替えます。
    - 差し替え後も、挙動はDay2で実装した `DeletedUserButton` と同じになるようにします。

### タスク4：ストーリーの追加実装

上記で実装した各コンポーネント（`CustomButton`、`CustomCard`、`CustomModal`）について、新しいストーリーを追加し、Storybookドキュメントを充実させます。

細かい指定はないため、コンポーネントを使う人が使い方を理解しやすいように、バリエーションを増やして実装します。

ストーリー追加に伴ってカスタムコンポーネントの内容修正が必要になる場合もあり得ますが、

その場合は講師に相談してから進めるように指示されています。このタスクについては言語化は

不要で、作業時間の見積もり報告のみが必要です。

### タスク5：アプリケーションへの統合

タスク1〜3で作成した共通コンポーネントをユーザー管理システムに反映し、UI/UXを向上させます。

1. **タスク5-1：コンポーネントの反映**
    - カードやボタンを差し替えたり、削除時にモーダルを表示させたりして、作成したカスタムコンポーネントを使用します。
    - このタスクは言語化不要で、対象箇所の確認と見積もり報告のみが必要です。
2. **タスク5-2：UI/UXを向上させる**
    - ユーザー一覧画面内に以下の機能を追加します。
        - **ユーザー検索機能**: ID検索と役職検索に対応させます。フリー検索ではなく、プルダウン形式で実装します。
        - **ユーザー並び替え機能**: IDの昇順、降順で並び替えができるようにします。並び替え用のタグの形式は任意です。
    - このタスクについては言語化を行います。

Day3は、共通コンポーネント化とその活用、そして既存画面への機能追加によるUI/UX改善が主な焦点となるようです。各コンポーネント作成時には、Storybookとの連携や、提供されたコードのTODO箇所の修正が重要なポイントとなります。
