# データベース基礎_台本

皆さん、これまでの学習で、WEBサービスの仕組み、クライアント（皆さんのReactアプリ）とサーバー間の通信手段であるWEB APIについて学んできました。

特に、WEB APIを使うことで、サーバーからデータを取得したり、サーバーにデータを送ったりできることが分かりました。

本日は、そのデータベースの基礎と、実際にデータを操作する方法について学習します。

本日の学習内容は以下の通りです。

- **データベースの役割**

資料の該当箇所を確認しながら進めていきましょう。

### 1. データベースの役割

資料にもあるように、皆さんがReactを使って体験したデータの登録や取得、更新、削除といった操作、これらをまとめて「**CRUD処理**」と呼びました。

- C: **Create** (作成)
- R: **Read** (読み込み/取得)
- U: **Update** (更新)
- D: **Delete** (削除)

実は、このCRUD処理は、**データベース上のデータを操作するための仕組み**なのです。

前回の学習ではWEB APIを意識して操作していましたが、今回はそのデータの本体を管理している**データベース自体を操作**してみましょう。

### 2. データベースツールのインストールと準備

データベースを実際に操作するためには、データベース管理システムが必要です。

今回は資料にある「PostgreSQL」という代表的なデータベース管理システムとその管理ツールである「pgAdmin」を利用します。

資料の**「ツールのインストール方法」**を参照しながら、

以下の手順でインストールを進めてください。

1. 指定されたURLからインストーラーをダウンロードします。Windows x86-64の最新版を選択しましょう。
2. インストーラーを起動し、基本的に「Next」をクリックして進めます。
3. インストールするツールは初期設定のままで問題ありません。
4. データの保管場所も変更不要です。
5. **パスワード**を設定します。これは非常に重要なので、**必ずメモ**しておいてください。
6. ポート番号も初期値のままでOKです。
7. 地域設定は「Japanese, Japan」を選択します。
8. 設定確認後、インストールを開始します。

インストールが完了したら、次にデータベースの準備をします。資料の**「pgAdminの利用」**を参照してください。

1. Windowsのアプリケーションから「pgAdmin」を起動します。
2. **初回起動時**にもパスワード設定が求められるので、**任意のパスワードを設定し、これもメモ**しておいてください。
3. 画面左側の「Servers」をクリックすると、設定したパスワードの入力が求められるので、入力してOKをクリックします。
4. 認証が完了したら、まずはデータベース自体を作成します。「Database」の項目で右クリックし、「Create」 > 「Database」を選択します。
5. 表示されたウィンドウの「Database」入力欄に `tutorial` と入力し、「Save」をクリックします。左側に `tutorial` が追加されることを確認します。
6. 次に、データを格納する「テーブル」を作成します。`tutorial` を展開し、「Schemas」 > `public` と進み、「Tables」にカーソルを合わせて右クリックし、「Create」 > 「Table」を選択します。
7. 「Create Table」画面が表示されるので、「Name」欄に `users` と入力します。
8. 「Columns」タブをクリックし、テーブルにどのような項目（カラム）を持たせるかを設定します。カラムはReactでいう**Interface**のようなもので、データの型や名前を定義します。
9. 「＋」ボタンを3回クリックし、資料の画像のように、`id`、`name`、`age` というカラム名と、対応するデータ型などを設定します。細かい内容は一旦気にせず、画像通りに入力し、「Save」をクリックしてください。

ここまでの操作が問題なく済んでいれば、今回学習するデータベースの準備は完了です。

### 3. データベースを操作する命令文（SQL）

準備したデータベースに対して、実際にデータを登録したり取得したりするには、特別な「命令文」を使います。これらの命令文を「**SQL文**」または「**クエリ**」と呼びます。

データベースでよく利用される操作と、それに対応するSQL命令文は以下の4つです。

- **挿入/登録**: `INSERT` (CRUDのCreateに対応)
- **選択/取得**: `SELECT` (CRUDのReadに対応)
- **更新**: `UPDATE` (CRUDのUpdateに対応)
- **削除**: `DELETE` (CRUDのDeleteに対応)

これらの命令文を使って、データを操作していきましょう。

### 4. データ操作の実践 (Query Tool)

SQL文を実行するためには、pgAdminの「Query Tool」を利用します。

pgAdminの左上にある「Tools」タブを選択し、その中の「Query Tool」をクリックしてください。空白のエディタが表示されます。ここにSQL文を記述していきます。

### データを登録する (INSERT)

まずはデータを `users` テーブルに登録してみましょう。
エディタに以下の命令文を記述し、実行ボタンを押してください。

```
INSERT INTO users (name, age) VALUES ('yamada', 22);

```

実行に成功したかどうかは、下部のメッセージ欄に表示されます。成功すれば完了です。

続けて、複数のデータを一度に登録してみましょう。
**注意点**として、資料にもあるように、**Query Toolのワークスペースに複数の命令文があると、全て実行されてしまいます**。先ほど記述した `INSERT` 文は削除するか、 `ctrl + /` キーを押してコメントアウトしておきましょう。

コメントアウトしたら、以下の命令文を記述し、実行してください。

```
INSERT INTO users (name, age) VALUES ('tanaka', 30), ('sato', 33);

```

これで `tanaka` さんと `sato` さんのデータが一度に登録されます。

**やってみよう**: 任意の名前と年齢を入力して、追加で2人分のデータを登録してください。登録すると全部で5人分のデータが登録された状態になるはずです。

### データを取得する (SELECT)

次に、登録したデータを取得してみましょう。
**INSERT文は必ず削除するかコメントアウト**してください。

以下の命令文を記述し、実行してみてください。

```
SELECT name, age FROM users;

```

実行すると、先ほど登録したデータが**テーブル形式で一覧表示**されます。`name` と `age` という指定したカラムだけが表示されていますね。

資料にあるように、皆さんがWEB APIを使ってデータを取得する際、サーバー側ではこのように**SELECT文が実行されている**ことがあります。

特定の条件でデータを絞り込んで取得することも可能です。資料にあるクエリを見てみましょう。

```
SELECT * FROM users WHERE age >= 30;

```

このクエリでは、いくつか新しいキーワードが登場しています。

- : これは**対象のテーブルにあるカラムを全て指定する**という意味です。今回の `users` テーブルであれば、 `id`, `name`, `age` の全てを意味します。もし `SELECT name FROM users;` とすれば、名前のカラムだけを取得できます。
- `WHERE`: これは**絞り込む条件を加える**ための文法です。Reactで使った `if` 文に近いイメージです。
- `age >= 30`: これが具体的な条件です。「`age` カラムの値が30以上」という条件を指定しています。

つまり、このクエリは「`users` テーブルから、年齢が30歳以上のユーザーの全てのカラム情報を取得する」という意味になります。

比較演算子として、 `>=` 以外にも、 `>` (より大きい), `<` (より小さい), `<=` (以下), `=` (同じかどうか) などがあります。

さらに、資料にあるように「かつ (`AND`)」や「または (`OR`)」といった条件を組み合わせることも可能です。

```
SELECT * FROM users WHERE name = 'yamada' AND age >= 30;

```

このクエリは、「`users` テーブルから、名前が 'yamada' **かつ** 年齢が30歳以上のユーザーの全てのカラム情報を取得する」という意味になります。このように条件を組み合わせることで、特定のデータだけを正確に取得することができます。

**やってみよう**: 以下の条件に合うユーザー情報を取得するクエリを実際に書いて、実行してみてください。

- 年齢が20歳以上のユーザー情報
- 名前がsatoのユーザー情報
- IDが4のユーザー情報 (カラムに `id` が追加されていることを確認してください)
- 年齢が25歳以上で32歳以下のユーザー情報

データの更新についてですが、そのあとの削除も同様で**id等を指定して全件が対象にならないようにする**というのを覚えておいてください。

### データを更新する (UPDATE)

次に、既存のデータを更新してみましょう。
以下のクエリを記述し、実行します。

```
UPDATE users SET age = 21 WHERE id = 3;

```

このクエリも、なんとなく意味が読み解けるのではないでしょうか。

- `UPDATE users`: `users` テーブルのデータを更新します、という意味です。
- `SET age = 21`: `age` カラムの値を 21 に設定します、という意味です。
- `WHERE id = 3`: `id` カラムの値が 3 であるデータを対象とします、という意味です。

**データを更新する場合、更新する対象を `WHERE` で指定することが非常に重要**です。もし `WHERE` 句を忘れて `UPDATE users SET age = 21;` のように実行してしまうと、なんと**テーブル内の全てのユーザーの年齢が21歳に更新されてしまう**という恐ろしい結果になります。必ず対象を指定しましょう。

**やってみよう**: 資料にある以下の内容を実際にやってみてください。

- すでに登録されているデータの名前を変更してみましょう。
- すでに登録されているデータの年齢を変更してみましょう。
- 追加で2~3人の情報を年齢が0の状態で登録し、登録後にUPDATEを利用してそれぞれ別々の年齢に変更してみましょう。

### データを削除する (DELETE)

最後に、データを削除する方法について学習します。

まずは基本的な `DELETE` 文を見てみましょう。
以下のクエリを記述し、実行します。

```
DELETE FROM users WHERE id = 5;

```

形式は `UPDATE` 文に似ていますね。

- `DELETE FROM users`: `users` テーブルからデータを削除します、という意味です。
- `WHERE id = 5`: `id` カラムの値が 5 であるデータを対象とします、という意味です。

`UPDATE` と同様に、**削除する場合も `WHERE` 句で対象を明確に指定**する必要があります。

もし対象を指定しないで `DELETE FROM users;` のように実行すると、これもなんと**テーブル内の全てのデータが削除されてしまう**という結果になります。注意してください。

さて、資料にはもう一つ別のクエリが紹介されています。

```
UPDATE users SET deleted = true WHERE id = 5;

```

このクエリ、どこかで見た覚えはありませんか？
前回のWEB APIを使ったユーザー情報削除の際に、「削除したことにする」というデータ操作を体験したことを覚えていますか？

データベースにおいても、実際に `DELETE` 文を使って物理的にデータを削除するのではなく、**`UPDATE` 文を使って「削除フラグ」のような専用のカラム（例: `deleted`）の値を更新し、「消したことにする」という方法が圧倒的に多いのです。これを論理削除**と呼びます。

なぜ物理削除（`DELETE`）をあまり使わないのでしょうか？資料にあるように、一度 `DELETE` で削除してしまうと、**簡単にデータを元に戻すことができません**。

バックアップなどがなければ完全にデータが失われます。完全に消えてしまうと後で困るケースが多いため、基本的には論理削除が一般的であると覚えておきましょう。

### 5. まとめ

本日は、WEBサービスを支える重要な要素である「**データベース**」について学習しました。

データベースは**大量のデータを効率的・安全に管理**するためのツールであり、そのデータを操作するためには**SQL文（クエリ）**を使います。

特に、データの「登録（`INSERT`）」、「取得（`SELECT`）」、「更新（`UPDATE`）」、「削除（`DELETE` または `UPDATE` による論理削除）」といった**CRUD操作**は、WEBアプリケーション開発においてデータベースと連携する上で不可欠な知識です。

今回学習したSQLの基礎を理解することで、皆さんが今後開発するアプリケーションで、より複雑なデータの管理や取得、更新などを実現できるようになります。
