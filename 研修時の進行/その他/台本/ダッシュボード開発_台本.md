# ダッシュボード開発_台本

前回の学習を踏まえまして、今回は具体的な開発プロジェクトについて見ていきたいと思います。

お手元にある「ダッシュボード画面を開発する.pdf」という資料をご覧ください。この資料では、まさに皆さんがこれから取り組む**ダッシュボード画面の開発**について、その準備から具体的なタスク、そしてチームでの進め方までが詳しく示されています。

まず、開発を始めるにあたっての**準備**です。皆さんはSlackで共有された対象のリポジトリを、まずお手元のコンピューターに**クローン**します。GitとGitHubで学んだように、リモートリポジトリの内容をローカルに取り込む作業ですね。その後、開発環境の構築を行い、`npm run dev` というコマンドを実行して、**画面が正常に表示できることを確認**します。これで、開発のスタートラインに立つことができます。

さて、このプロジェクトで皆さんが行うべき**タスク**はいくつかあります。

資料には以下のタスクが挙げられています:

- ユーザー統計情報のグラフ作成
- 売上データの折れ線グラフ作成
- ダッシュボードの修正（作成したグラフの配置など）
- 商品一覧ページの追加
- 商品カテゴリ別の割合グラフ作成
- 商品別の売上グラフ作成

これらのタスクをチーム内で分担して進めていくことになります。また、チームの中から**1名がリーダーとなり、プルリクエストの管理**を行う役割を担います。

それでは、それぞれの**タスク詳細**について具体的に見ていきましょう。

まず**ユーザー統計情報のグラフ作成**です。これは、ダッシュボード上にユーザー統計情報の**棒グラフ**を表示するタスクです。作業としては、`src/components/` ディレクトリの中に `UserStatsChart.tsx` というファイルを作成します。グラフの実装には、**Rechartsライブラリを使用**します。データは、モックデータである `userData.ts` を利用し、月ごとの増加数やアクティブ数などをグラフに表示するイメージです。

次に、**売上データの折れ線グラフ作成**です。こちらもダッシュボード上に表示されるグラフで、**折れ線グラフ**を作成します。作業内容は、`src/components/` に `SalesChart.tsx` ファイルを作成し、Rechartsライブラリを使って実装します。データは `salesData.ts` というモックデータを利用します。特徴的なのは、**日別、週別、月別でデータを切り替えて表示する機能も実装する**点です。

そして、作成したグラフなどをダッシュボードに表示する**ダッシュボードの修正**タスクがあります。`Dashboard.tsx` ファイルの中で、元々あるサンプルの `PlaceholderChart` というコンポーネントを削除し、代わりに先ほど作成した `UserStatsChart` と `SalesChart` をインポートして配置します。さらに、後述する `CategoryPieChart.tsx` と `SalesPieChart.tsx` もインポートして表示する作業も含まれます。

次に、**商品一覧ページの追加**というタスクです。これは、商品のリストを**テーブル形式で表示するページ**を作成します。`src/pages/` ディレクトリに `Products.tsx` というファイルを作成します。テーブルに表示するための商品のモックデータも作成します。データは `productData.ts` というファイル名で、`id`、`name`、`category`、`price` といった情報を持つ `Product` というインターフェースの配列として、**6種類以上の商品データ**を追加します。作成した商品一覧ページへの**ルーティング**と、画面左側にある**サイドメニューからの遷移**も実装する必要があります。

さらに、**商品カテゴリ別の割合を示す円グラフ**を作成します。

これもダッシュボード上に表示されます。`src/components/` に `CategoryPieChart.tsx` ファイルを作成し、Rechartsライブラリで実装します。

データは `productData.ts` を利用しますが、商品データとは別に `categoryData` という名前で、

カテゴリ名と割合を示す数値（`value`）を持つデータを**6つ以上追加**して利用します。

最後に、**商品別の売上金額を示す円グラフ**を作成します。

こちらもダッシュボードに表示される円グラフです。`src/components/` に `SalesPieChart.tsx` ファイルを作成し、Rechartsライブラリで実装します。利用するデータは、売上データである `salseData.ts` のモックデータです。

これらのグラフ作成においては、**Rechartsの公式ドキュメント**が参考に載っています。

また、画面の**レイアウト**に関しては、**MUI（Material UI）の公式ドキュメント**を参考に進めることが示されています。

使用するデータは、ほとんどが `src/data` ディレクトリに格納されているモックデータを利用します。

完成後の**ページ構成**としては、主に**ダッシュボードページ (`/`)** と**通知センターページ (`/notifications`)** があります。ダッシュボードにはサイドメニューとヘッダーが表示され、メインエリアにユーザー統計グラフと売上グラフが配置されます。ヘッダーにはテーマ切り替えスイッチもありますね。通知センターページでは、通知一覧が表示され、クリックすると詳細が表示されるという動作になります。

全体的な**動作イメージ**としては、**サイドメニューから各ページに移動できる**ようにナビゲーションが実装されます。ページ遷移は**React Router**で行われます。**テーマ切り替え**機能では、ダークモードとライトモードを切り替えることができ、選択したテーマがアプリ全体に適用されます。グラフはユーザー統計と売上データが視覚的に表示され、売上データは期間を切り替えて表示できる点がポイントです。通知センターでは、クリックで詳細が表示されるという動作になります。

さて、これらのタスクをチームでどのように進めるか、その**作業の流れ**が示されています。

まずはじめに、プロジェクトをクローンして環境構築を行います。そして、メンバーの中から**一人のリーダーを決定**します。リーダーは、今後の作業分担の調整、各作業内容の**マージ**、そして完了報告といった役割を担います。

次に、**Gitのルール**についてです。

プロジェクトをクローンした後、リーダーがチーム用の**共通ブランチ**を作成します。

ブランチ名は `team-[チーム名]-[入社日]` という形式で、チーム名はリーダーの苗字の頭文字、

入社日は数字4桁です。

例としては `team-y-1001` のようになります。リーダーはこのブランチを作成したらリモートリポジトリに**プッシュ**し、他のメンバーはそれを `fetch` して `merge` し、ローカルリポジトリに反映させます。この共通ブランチを作業の集約場所として運用します。

**作業を開始する**際には、このチーム用の共通ブランチから、**自分自身の作業用ブランチを作成**します。ブランチ名は `team-[チーム名]-[入社日]_自分の名前` のように、共通ブランチ名にアンダースコアと自分の名前を付け加えます。

ブランチを作成したらリモートリポジトリに反映し、割り振られた機能開発を進めます。

**機能を統合する**段階では、自分の担当機能が完成したら、**GitHub上で**自分の作業用ブランチから**チーム用ブランチへマージリクエスト**を行います。

そしてリーダーにそのことを共有し、リーダーが内容を確認して**マージ**を行います。こうして、すべての機能が共通ブランチにマージされ、問題なく動作すればタスク完了となります。

このように、この資料にはダッシュボード画面という具体的な開発対象と、それをチームで開発するためのGit/GitHubを活用した具体的なプロセスが示されています。この流れに沿って作業を進めることで、効率的に開発を進めることができるでしょう。
