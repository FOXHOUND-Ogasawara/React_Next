### 1. WEB APIとは

**WEB API**は、簡単に言うと**インターネット上に公開されている便利なメソッドのようなもの**です。WEBエンジニアの間では単に**API**と呼ばれることも多いです。

WEB APIを利用すると、**URLにアクセスする**ことで、特定のサーバーから簡単に**データを取得したり**、**サーバーの機能を利用したり**することができます。

皆さんはReact課題で `fetch` というものを使って、**URLにアクセスした経験**があるかと思いますが、あれも**WEB APIを「叩いて」（利用して）データを取得していた** わけです。

この「叩いて」という表現もエンジニア特有なので覚えておきましょう。

世の中には様々なWEB APIが存在します。

今回の研修では、猫の画像を取得できるAPI （通称：猫ちゃん）や、天気情報を取得できるAPI （通称：お天気）などがあります。

これらのAPIを使うことで、ゼロからすべての機能を作るのではなく、**既存の便利な機能やデータを利用して、効率的にアプリケーション開発を進めることができます。**

WEBエンジニアとして働く上で、このWEB APIの存在は**必ず登場**します。

そして、**WEB APIは人が作ったもの**なので、皆さんも**自分でAPIを作る立場になる可能性**も

十分にあります。

使う側、そして作る側としても理解しておくべき非常に重要な技術です。

### 2. WEB APIのドキュメントを読み解く

WEB APIを利用するためには、そのAPIが「**どう使えばいいか**」を知る必要があります。

そのためには、**APIの提供元が公開している「公式ドキュメント」を確認する**のが最も重要です。

今回の資料では、「The Cat API」と「OpenWeather」という二つのAPIを例に進めます。

まずはこれらの公式ドキュメントにアクセスし、内容を確認してください。

ドキュメントを確認する上で、特に**注目すべきポイント**がいくつかあります。

これらは、前回の座学で学習したリクエストの構成要素と強く関連しています。

- **URL（エンドポイント）**: どのURLにアクセスすれば、目的の機能やデータに到達できるのか。APIの機能ごとにURLが分かれていることが多いです。
- **HTTPメソッド**: そのURLに対して、どのようなHTTPメソッド（GET, POST, PUT, DELETEなど）を使えば良いのか。データを取得したいのか（GET）、何かを登録したいのか（POST）、更新したいのか（PUT）などで使うメソッドが異なります。
- **送信する情報**: リクエストを送る際に、どのような情報をサーバーに渡す必要があるのか。
例えば、天気APIであれば知りたい「地域名」などです。
これらの情報は、**リクエストヘッダー**や**リクエストボディ**、または**クエリパラメータ**として送信します。

### 具体例：The Cat APIのドキュメントを読む

資料には、The Cat APIのドキュメントにあるJavaScript (`fetch`) のコード例が記載されていますね。

```
const headers = new Headers({
  "Content-Type": "application/json",
  "x-api-key": "DEMO-API-KEY"
});
var requestOptions = {
  method: 'GET',
  headers: headers,
  redirect: 'follow'
};
fetch(
"https://api.thecatapi.com/v1/images/search?size=med&mime_
requestOptions
)
  .then(response => response.text())
  .then(result => console.log(result))
  .catch(error => console.log('error', error));

```

このコードを、前回の座学で学習したHTTPリクエストの形に分解してみましょう。

- **HTTPメソッド**: `method: 'GET'` とあるので、**GETメソッド**を使用しています。
- **URL（エンドポイント）**: `fetch` 関数の中に書かれている `"https://api.thecatapi.com/v1/images/search?...` という部分がアクセス先の**URL**です。
特に、`https://api.thecatapi.com/v1/images/search` がエンドポイントとなります。
- **ヘッダー**: `headers` というオブジェクトに設定されている `Content-Type` や `x-api-key` が
**リクエストヘッダー**となります。
APIによっては、認証のために `x-api-key` のような専用のキーをヘッダーに含める必要があったりします。
- **送信する情報（クエリパラメータ）**: URLの `?` 以降に `size=med&mime_types=jpg&format=json&has_breeds=true&order=RANDOM&page=0&limit=1` という
文字列が付いていますね。
GETリクエストの場合、このように**URLに直接データを追加して送信**します。
この部分を「**クエリパラメータ**」と呼びます。
    - クエリパラメータは、URLの末尾に `?` を付けてから記述を開始します。
    - データは `key=value` の形式で記述します。値にクォーテーションは付けません。
    - 複数のパラメータがある場合は `&` 記号で繋げます。
    - この例では、「サイズはmed」、「jpeg形式」、「JSON形式」、「犬種情報あり」、
    「ランダムな順序」、「0ページ目」、「1件だけ取得」という条件を指定して
    猫の画像を検索するリクエストを送っていることになります。

**注意点**として、前回の座学で説明した通り、**GETメソッドではデータをリクエストボディではなくURL（クエリパラメータ）として送信する** のが一般的です。
一方、POSTやPUTなどではリクエストボディにデータを含めます。

- **レスポンス**: このAPIにリクエストを送ると、**JSON形式**で猫の画像の情報が返ってきます。
ドキュメントにはレスポンスの例も載っているので、どのようなデータが返ってくるか
確認できます。
例えば、画像のURLや高さ、幅などの情報が含まれています。

このように、APIドキュメントには、**どのURLに、どのHTTPメソッドでアクセスし、
どんな情報をどんな形式で送れば、どんなデータが返ってくるのか**、というAPIの利用方法が
詳しく書かれています。
APIを利用する際は、まずこのドキュメントをしっかりと読み込むことが成功の鍵となります。

### 3. API Testerを使ってみる

APIドキュメントを確認したら、実際にAPIにリクエストを送って、期待通りのレスポンスが
返ってくるか試してみましょう。
Reactアプリケーションに組み込む前に、手軽にAPIの動作を確認できる便利なツールがあります。それが「**API Tester**」です。

API Testerは、Chromeの拡張機能として提供されており、資料にインストール方法が記載されています。

インストールして開いてみると、非常にシンプルな画面であることが分かります。
使い方は簡単で、ドキュメントで確認した以下の情報を入力するだけです。

- **HTTPメソッド** (GET, POSTなど)
- **URL** (エンドポイントとクエリパラメータ)
- **HEADER** 欄に必要な情報 (API Keyなど)
- **BODY** 欄に送信するデータ (POSTなどの場合)

これらの情報を入力したら、「Send」ボタンをクリックするだけで、実際にAPIにリクエストを
送信し、サーバーからの**Response**を確認することができます。
天気APIを叩いて正しくレスポンスが返ってくるかやってみましょう。

～恐らくAPI KEYで苦戦する（401エラーが返ってくる）～

先程の猫ちゃんでは必須ではなかったのですが、お天気の方はAPI KEYが必須になります。

API KEYというのが発行されているのでそれを使用してみてください。

### 4. Reactを使ってAPIを利用する

APIドキュメントを読み込み、API Testerで動作確認ができたら、いよいよ皆さんのReactアプリケーションからAPIを利用する段階です。

資料には、今回紹介した二つのAPI（お天気情報、猫ちゃん）をReactから利用し、
取得したデータを画面に表示するところまでチャレンジしてみましょう。

- **猫ちゃんの仕様**: 猫の画像が表示され、ボタンを押すと別の猫の画像に切り替わる。
- **お天気の仕様**: 地域を選択すると、その地域の天気情報（アイコン、温度）が表示される。

これらの機能を実装するために、皆さんはReactコンポーネントの中でJavaScriptの `fetch` 関数などを使い、学習したWEB APIのURLに対して、適切なHTTPメソッドやヘッダー、クエリパラメータを指定してリクエストを送信することになります。
サーバーから返ってきたレスポンスデータ（JSON形式）をJavaScriptで処理し、
画面に表示するようにコーディングを進めていくことになります。

### 5. まとめ

本日は、WEBサービスの重要な要素である「**WEB API**」について学習しました。
APIは、**クライアントとサーバーがインターネット上でデータをやり取りするための規約**であり、**URLアクセスとHTTP通信**（リクエスト/レスポンス、メソッド、ヘッダー、ボディ、クエリパラメータ） を使って行われます。
APIを利用する際は、まず**公式ドキュメント**をしっかりと読み込み、API Testerなどのツールで動作を確認する ことが重要です。

これらの知識は、皆さんが今後Reactアプリケーションでサーバーと通信し、動的なコンテンツを表示したり、ユーザーの操作に応じてデータを更新したりするために**不可欠**となります。
