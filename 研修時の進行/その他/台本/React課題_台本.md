# React課題_台本

皆さん、こんにちは！

本日は、これまで学んできたReactの知識を実践的に活用する「React課題」に取り組みます。

この課題を通じて、WEBサービスの根幹をなすデータの操作、特に表示や絞り込み、

そしてショッピングカート機能の実装を体験し、実践的なスキルを身につけていきましょう。

### 1. 環境構築を始めましょう

まずは開発環境の準備からです。
事前学習で利用した「Reactではじめるプログラミング基礎コース」の資料を参考に、

支給されたPCに必要な環境を構築してください。

SLスタジオで利用していたアカウントをそのまま使用できます。

NVMの設定（nvm use `<バージョン>`まで）はSLスタジオ側で実施する内容となりますので、そこまで進めてください。

合わせて、VSCodeの拡張機能も必ずインストールしましょう。
似たものが複数ある場合があるので、資料の画像を参考にしながら、

以下の拡張機能を選んでインストールしてください。

- **Typescript React code snippets**: 予測変換を出してくれるプラグインです。
- **Prettier**: プログラムコードを整形し、読みやすく整理してくれます。
- **ESLint**: 書き方が間違っている、または良くない書き方をしている場合に指摘してくれます。
- **Path Intellisense**: ファイルを指定する際に、パスのヒントが表示されます。

インストール後は、VSCodeに反映させるために再起動が必要です。
上記の拡張機能を含め、VSCodeでReactが実行できる環境を構築してください。

プロジェクトを作成する際は、Cドライブ内に専用のフォルダを作成し、その配下に作成するようにしましょう。

VSCodeでPowerShellを使用する際に「npm create vite@latest」と打つと

エラーが起きることがあります。

その際は、資料に記載されている手順に従ってPowerShellの権限設定を変更してください。

### 2. プロジェクトの準備

環境構築が完了したら、プロジェクトの準備に移ります。
まず`npm create vite@latest`コマンドを使用して新しいReactプロジェクトを作成してください。プロジェクト名は **「products-management-system」** とします。

次に、このプロジェクトで必要となるライブラリをインストールします。

資料に記載されている以下のコマンドをコピー＆ペーストして実行してください。

```
npm install react-router-dom @mui/material @mui/icons-material @emotion/react @emotion/styled

```

これらのライブラリは、ルーティング（react-router-dom）や

UIコンポーネント（MUI関連、emotion関連）のために利用します。

ライブラリのインストール後、以下のフォルダとファイルを `src` ディレクトリ配下に作成してください。

- `components` フォルダ
    - `ProductList.tsx`
    - `ProductItem.tsx`
    - `Cart.tsx`
    - `CartItem.tsx`
- `pages` フォルダ
    - `Home.tsx`
- `types` フォルダ
    - `index.ts`

フォルダとファイルが作成できたら、資料に提供されているコードをそれぞれの該当ファイルに

コピー＆ペーストしてください。

App.tsx、ProductList.tsx、ProductItem.tsx、Cart.tsx、CartItem.tsx、Home.tsx、index.ts のコードはSlackに送っているのでそれを使用してください。

ファイル数が多いため、コピーするファイルを間違えないように注意して進めましょう。

typesフォルダに作成した `index.ts` には、商品データの型定義として `Product` インターフェースが定義されています。

これは、取得する商品データの構造を定義したものです。

### 3. タスク内容の確認

プロジェクトの準備が整ったら、いよいよ実装に入ります。

この課題では以下の4つのタスクに取り組みます。

- コンポーネントの役割分担としては、Home.tsxがページ全体、データ取得、state管理、
ProductList.tsxがリストの生成、ProductItem.tsxが個々の商品表示とカート追加ボタンの
設置（タスク3）を担当します。
- **タスク1: 商品データの表示機能を実装する**
    - 外部API（`https://fakestoreapi.com/products`）から商品データを取得し、
    Home.tsxでデータの取得処理とstate管理を実装します。
    - 取得した商品データを利用して、ProductList.tsxで商品リストを表示する処理を実装し、
    ProductItem.tsxで個々の商品（タイトル、価格、画像）を表示する処理を実装します。
- **タスク2: カテゴリーによる絞り込みを実装する**
    - カテゴリーAPI（`https://fakestoreapi.com/products/categories`）からカテゴリーデータを取得し、Home.tsxで取得処理とstate管理を実装します。
    - カテゴリー選択用のプルダウンメニューを実装し、選択されたカテゴリーに応じて
    商品リストが更新されるように、フィルタリング処理を追加します。
- **タスク3: カート機能を実装する**
    - 商品をカートに追加したり、カートから削除したり、合計金額を計算する機能を
    実装します。
    - App.tsx内でカートの状態管理（`cartItems` state）を行います。
    - ProductItem.tsxに「カートに追加」ボタンを実装し、クリック時に商品を
    カートに追加する `addToCart` 関数（App.tsxで定義しpropsとして渡す想定）を
    呼び出します。
    - Cart.tsxとCartItem.tsxに処理を追記して、カートの内容表示、商品の削除、
    合計金額の計算を完成させます。

ここまでのタスクを必須としてやってもらいます。

期限等はSlackに共有します。

まずは、フォーマットに沿ってタスクの言語化を進めていってください。

- **タスク4: ローディングを実装する（追加課題）**
    - データの取得完了までの間にローディング表示を追加します。
    - Home.tsxにローディング状態を管理するstateを追加し、データ取得中はMUIの `CircularProgress` コンポーネントを表示するように実装します。

これらのタスクを順番に進めていくことで、Reactを使った基本的なWEBアプリケーション開発の流れを体験できます。
