# ユーザー管理システム_Day2_台本

提供された資料「ユーザー管理システム_day2.pdf」に基づき、Day2のタスクについて詳しくご説明します。

資料によると、Day2では主に以下の機能の実装が計画されています:

- **ユーザー一覧機能の実装**
- **ユーザー削除機能の実装**
- **ユーザー詳細機能の実装**
- **追加機能の実装** (フォームバリデーション強化)

これらの各機能には、いくつかの細かいタスクが定義されています。以下にそれぞれ詳細を記載します。

### ユーザー一覧機能関連のタスク

ユーザー一覧機能では、以下のタスクを行います:

1. **タスク1-1：ユーザー一覧コンポーネントの作成**
    - 対象ファイルは `components/UserList.tsx` です。
    - 仕様としては、ユーザー一覧を表示するコンポーネントを作成します。
    - `UserListProps` インターフェースが定義され、`users: User[]` を受け取ります。
    - コンポーネント名は `UserList` です。
    - 受け取ったユーザー情報を利用して `UserCard` コンポーネントを呼び出します。
    - **注意点**として、`UserList` コンポーネント内でAPI (`fetchUsers`) を利用したデータ取得処理は記載しません。
2. **タスク1-2: UserListのストーリー作成**
    - 対象ファイルは `components/UserList.stories.tsx` です。
    - `UserList` コンポーネントのStorybookストーリーを作成し、コンポーネントのドキュメント化を行います。
    - 仕様として、`Meta` を使用してメタデータを定義し、`StoryObj` を使用してストーリーを定義します。
    - デフォルトストーリーには例となるユーザーデータを設定し、実行確認ができるようにします。
3. **タスク1-3：ユーザー一覧画面の修正**
    - 対象ファイルは `app/users/page.tsx` です。
    - 作成した `UserList` コンポーネントを既存のユーザー一覧画面に反映します。
    - 仕様として、一覧ページにアクセスした際にユーザーの一覧が表示されるようにします。
    - 表示すべきユーザー情報を取得し、取得したデータを利用して `UserList` コンポーネントが表示されるようにします。

### ユーザー削除機能関連のタスク

ユーザー削除機能では、以下のタスクを行います:

1. **タスク2-1-1：削除用の新規関数追加**
    - 対象ファイルは `utils/api.ts` です。
    - **論理削除**用に新たな関数を作成します。
    - 仕様として、削除済みユーザーを取得しない処理が追加されます。
    - 対象箇所については質問、確認することが推奨されています。
    - 論理削除用に新規関数が追加され、関数名は任意とします。
2. **タスク2-1-2：ユーザー削除ボタンのコンポーネント作成**
    - 対象ファイルは `components/DeleteUserButton.tsx` です。
    - ユーザーを削除するためのボタンコンポーネントを作成します。
    - 仕様として、`DeleteUserButtonProps` インターフェースが定義され、`userId: number` と再レンダリング用の `onDelete: (userId: number) => void` を受け取ります。
    - コンポーネント名は `DeleteUserButton` です。
    - コンポーネントは `DeleteUserButtonProps` で定義した内容を受け取ることができます。
    - ユーザーの削除機能が実装され、削除は論理削除で行います。
    - 「削除」ボタンを押すことでデータが更新され、更新成功後は `onDelete` を実行して再レンダリングを行います.
    - **削除ボタンを押した際にconfirmを使って確認画面を表示させます**。
    - **ヒント**として、`if (confirm('本当にこのユーザーを削除しますか？')) {}` と `onClick` の使用が挙げられています。
3. **タスク2-2：DeleteUserButtonのストーリー作成**
    - 対象ファイルは `components/DeleteUserButton.stories.tsx` です。
    - `DeleteUserButton` コンポーネントのStorybookストーリーを作成し、コンポーネントのドキュメント化を行います。
    - 仕様として、`Meta` を使用してメタデータを定義し、`StoryObj` を使用してストーリーを定義します。
    - デフォルトストーリーに例となるユーザーのIDを設定します。
4. **タスク2-3：UserCardコンポーネントとDeleteUserButtonの統合**
    - 対象ファイルは `components/UserCard.tsx` および `components/UserList.tsx` です。
    - `UserCard` コンポーネント内の削除ボタンを差し替え、削除機能を統合します。
    - 仕様として、`UserCard` コンポーネント内の削除ボタンが `DeleteUserButton` になります。
    - 削除された後にユーザー一覧が再レンダリングされ、該当ユーザーが見えなくなるようにします。
    - **再レンダリングのポイント**として、以下の点が挙げられています:
        - `UserCard` コンポーネントに再レンダリング用の関数を受け取れるようにする。
        - `UserList` コンポーネントに表示用のデータを`state`で管理する。
        - 削除が実行されたら`state`が更新される関数を追加する。
        - `filter` を利用することで削除済みかを判定する。

### ユーザー詳細機能関連のタスク

ユーザー詳細機能では、以下のタスクを行います:

1. **タスク3-1：ユーザー詳細コンポーネントの作成**
    - 対象ファイルは `components/UserDetails.tsx` です。
    - ユーザー詳細情報を表示するコンポーネントを作成します。
    - 仕様として、`UserDetailsProps` インターフェースが定義され、`user: User;` を受け取ります。
    - コンポーネント名は `UserDetails` です。
    - コンポーネントは `UserDetailsProps` で定義した内容を受け取ることができます。
    - 以下のユーザー情報を表示します: ID、名前、メールアドレス、役職。
2. **タスク3-2：UserDetailsのストーリー作成**
    - 対象ファイルは `components/UserDetails.stories.tsx` です。
    - `UserDetails` コンポーネントのStorybookストーリーを作成し、コンポーネントのドキュメント化を行います。
    - 仕様として、`Meta` を使用してストーリーメタデータを定義し、`StoryObj` を使用してストーリーを定義します。
    - デフォルトストーリーに例となるユーザーデータが設定され、実行確認ができるようにします。
3. **タスク3-3：ユーザー詳細画面の作成と反映**
    - 対象ファイルは `app/users/[id]/details/page.tsx` および `app/components/UserCard.tsx` です。
    - `UserDetails` コンポーネントを使用してユーザー詳細画面の作成します。
    - 仕様として、コンポーネント名が `UserDetailsPage` です。
    - 表示すべきユーザー情報を取得します。
    - `UserDetails` コンポーネントをレンダリングし、詳細画面が表示されるようにします。
    - ページ全体のレイアウトが崩れていないことを確認します。
    - 正常に動作し、対象となるユーザー情報が確認できることを確認します。
    - **`UserCard` コンポーネントに「詳細」ボタンを追加し、遷移ができるようにします**。

### 追加機能の実装

上記のタスクが完了したら、チャレンジタスクとして以下の実装を行います:

1. **タスク4-1：フォームのバリデーションを強化する**
    - 既存のフォームコンポーネントに対して、バリデーションルールを追加します。
    - 仕様として、`react-hook-form` のバリデーション機能を活用して、フォームフィールドにルールを適用します。
    - わかりやすいエラーメッセージが表示されるようにします。
    - どのようなバリデーションを追加するかを自身で考え、方針に問題がないか確認してから実装を行います。

Day2では、これらのコンポーネント作成、Storybookによるドキュメント化、そして既存画面への機能統合などが主な作業内容となるようです。特にユーザー削除機能では、論理削除や再レンダリングの方法にポイントがあります。
