# Tips：曖昧なプロンプトで暴走するパターン10選

AI駆動開発研修における「AIが起こしやすい暴走・余分実装」パターンと、それに対する修正プロンプト例をまとめたテンプレートです。タスク実装時の参考や実習用課題に活用してください。

---

## 1. ローディング処理の勝手な追加

**暴走例**：`loading` state や "Loading..." 表示を追加

```tsx
const [loading, setLoading] = useState(true);
if (loading) return <p>Loading...</p>;

```

**修正プロンプト例**：

```
ローディング処理（state: loading, 表示: "Loading..."）は仕様外のため削除してください。
関連する state や条件分岐もすべて削除してください。

```

---

## 2. 不要な error state を追加

**暴走例**：

```tsx
const [error, setError] = useState(null);

```

**修正プロンプト例**：

```
error state は仕様に含まれていないため削除してください。

```

---

## 3. フォルダ・ファイル構成を勝手に変更

**暴走例**：コンポーネントを別フォルダに移動

```
src/components/CountryItem → src/features/Country/components/CountryItem

```

**修正プロンプト例**：

```
フォルダ構成やファイル位置の変更は仕様外のため元に戻してください。

```

---

## 4. 過剰な型定義や抽象化

**暴走例**：

```tsx
interface ExtendedCountry extends Country {
  id: string;
  isFavorite: boolean;
}

```

**修正プロンプト例**：

```
型定義は task1 で指定された項目のみ使用してください。
拡張フィールドの追加は禁止です。

```

---

## 5. population の勝手なフォーマット

**暴走例**：

```tsx
{country.population.toLocaleString()}

```

**修正プロンプト例**：

```
population の表示加工は仕様外のためプレーンな値で表示してください。

```

---

## 6. UIデザインの過剰実装

**暴走例**：

```tsx
<Card shadow="lg" radius="xl">...

```

**修正プロンプト例**：

```
UIデザインへの変更（影や余計な装飾）は仕様外のため削除してください。

```

---

## 7. 並び替え（sort）処理の勝手な追加

**暴走例**：

```tsx
countries.sort((a, b) => a.name.common.localeCompare(b.name.common));

```

**修正プロンプト例**：

```
タスク1では並び替え処理は不要のため削除してください。

```

---

## 8. 仕様外 state の追加

**暴走例**：

```tsx
const [selectedCountry, setSelectedCountry] = useState(null);

```

**修正プロンプト例**：

```
selectedCountry は task1 で対象外のため削除してください。

```

---

## 9. fetch を別関数・サービス層化する

**暴走例**：

```tsx
const fetchCountries = async () => { ... }

```

または `service/api.ts` などに分割

**修正プロンプト例**：

```
通信処理は Home.tsx の useEffect 内で行ってください。
関数分割やサービス層化は仕様外です。

```

---

## 10. インラインスタイルや CSS 処理の勝手な追加

**暴走例**：

```tsx
<div style={{ marginTop: '20px' }}>

```

**修正プロンプト例**：

```
スタイル変更は仕様に含まれていないため削除してください。

```

---

## 🔥 他に起こりやすい暴走追加例（補足）

| 暴走内容 | 対応策 |
| --- | --- |
| key にランダム値（Date.now()）を使用 | name.common など一意識できる値を使用させる |
| API URL を改変（fields 削除など） | 正しいエンドポイントに戻させる |
| useMemo / useCallback など premature optimization | 削除するよう指示 |
| Jest テストコードを生成 | 今回のタスク対象外と明記 |
