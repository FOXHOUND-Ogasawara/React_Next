### タスク2:タスク管理ボードのレイアウト作成

タスクをステータスごとに3列で表示するタスク管理ボードを作成する。

作業内容

- `src/components/TaskBoard.tsx` に、タスク管理ボード全体のレイアウトを実装する。
    - MUIの `Grid`（または `GridLegacy`）, `Box`, `Typography` を利用して、以下の3列レイアウトを作成する。
        - Todo列
        - In Progress列
        - Done列
    - `src/data/taskData.ts` で定義された `initialTasks` を読み込み、Reactの状態として保持すること。
        - 例：
            
            ```tsx
            import React, { useState } from 'react';
            import { initialTasks, Task } from '../data/taskData';
            // ↑Taskに警告文が出ることがあるが動作に支障はないのでそのまま進めてOK
            
            const [tasks, setTasks] = useState<Task[]>(initialTasks);
            ```
            
    - ステータスごとにタスクを絞り込む関数を定義する。
    - 上記の絞り込み結果を使って、各列に渡すタスク配列を作成する。
- `src/components/TaskColumn.tsx` に、1列分の表示レイアウトを実装する。
    - `TaskColumn` コンポーネントの props で、次の情報を受け取るようにする。
        - 列タイトル
        - ステータス
        - その列に表示するタスクの配列（`Task[]`）
    - MUI の `Paper`, `Typography`, `Box` を利用して、以下を表示する。
        - 列タイトル
        - タスク数（例：`Todo (3)` のように件数をカッコ付きで表示）
        - タスク一覧（この時点では、タスク名（`title`）のみ簡易表示でよい）
    - タスクは `map` を使ってループ表示すること。
- `TaskBoard.tsx` では、用意された `TaskColumn` コンポーネントを利用して、3列すべてを並べること。
    - 例：`columns` 配列を用意し、`map` で3列分を `Grid` に配置する。
    - それぞれの列に対して、タイトル・ステータス・対応するタスク配列を渡す。

### タスク3:ドラッグ＆ドロップによるタスク移動の実装

dnd-kit を利用して、タスクカードをドラッグ＆ドロップで別の列へ移動できるようにする。

作業内容

- `src/components/TaskBoard.tsx` を編集し、ドラッグ＆ドロップのコンテキストを設定する。
    - `@dnd-kit/core` から `DndContext`, `DragEndEvent` をインポートする。
    - 既存のレイアウト（`<Grid>...</Grid>`）を `<DndContext>` コンポーネントでラップする。
    - `onDragEnd` ハンドラを実装し、ドラッグ終了時にタスクの `status` を更新する。
        - 移動したタスクの ID（`active.id`）
        - ドロップ先の列 ID（`over.id`）
        - この2つの値を使って、`tasks` の状態を更新する。
- `src/components/TaskColumn.tsx` を編集し、列をドロップ可能エリアにする。
    - `@dnd-kit/core` から `useDroppable` をインポートする。
    - `useDroppable({ id: status })` を呼び出し、返り値の `setNodeRef` を `Paper` コンポーネントに設定する。
    - `isOver` を利用して、タスクをドラッグ中に上に乗せたときの背景色などを変える（任意）。
- `src/components/TaskCard.tsx` を編集し、カードをドラッグ可能にする。
    - `@dnd-kit/core` から `useDraggable` をインポートする。
    - `@dnd-kit/utilities` から `CSS` をインポートし、ドラッグ中のカードに変形（transform）を適用する。
    - `useDraggable({ id: task.id })` を呼び出し、返り値の
        - `setNodeRef`
        - `attributes`
        - `listeners`
        - `transform`
        - `isDragging`
- 動作確認
    - タスクカードをドラッグし、別の列にドロップすると、その列にカードが移動すること。
    - ブラウザをリロードすると、元のモックデータの状態に戻ること（状態はメモリ上のみで良い）。

💡**ヒント**

今回は書き方をヒントとして載せておきます。

書きながら「何をしているコードなのか」を考えてみましょう。

`DndContext` の書き方例

```tsx
import { DndContext, type DragEndEvent } from '@dnd-kit/core';

const handleDragEnd = (event: DragEndEvent) => {
  const { active, over } = event;
  if (!over) return;

  const taskId = String(active.id);
  const newStatus = over.id; // 'todo' など
  // setTasks で status を更新する
};

return (
  <DndContext onDragEnd={handleDragEnd}>
    {/* ここに <Grid> ... </Grid> を置く */}
  </DndContext>
);
```

`useDroppable` の書き方例

```tsx
import { useDroppable } from '@dnd-kit/core';

const { setNodeRef, isOver } = useDroppable({ id: status });

return (
  <Paper
	  ref={setNodeRef}
    sx={{
      background: isOver ? 'grey.100' : 'grey.50',
    }}
  >
    {/* タスク一覧 */}
  </Paper>
);
```

`useDraggable` の書き方例

```tsx
import { useDraggable } from '@dnd-kit/core';
import { CSS } from '@dnd-kit/utilities';

const { attributes, listeners, setNodeRef, transform, isDragging } =
  useDraggable({ id: task.id });

const style = {
  transform: transform ? CSS.Translate.toString(transform) : undefined,
  opacity: isDragging ? 0.7 : 1,
} as React.CSSProperties;

return (
  <Card
	  ref={setNodeRef}
    style={style}
    {...listeners}
    {...attributes}
  >
    {/* タスク情報 */}
  </Card>
);
```

### タスク5:タスクの絞り込み機能の追加

タスク管理ボードにフィルタ機能を追加し、**担当者名** や **優先度** で表示するタスクを絞り込めるようにする。

作業内容

- `src/components/TaskBoard.tsx` を編集し、絞り込み用の状態を追加する。
    - 以下の state を追加すること。
        - `filterAssignee`: `string` 型
            - 担当者名での絞り込み用
        - `filterPriority`: `'all' | 'low' | 'medium' | 'high'` 型
            - 優先度での絞り込み用
    - 初期値は以下のようにする。
        - `filterAssignee`: `''`（空文字）
        - `filterPriority`: `'all'`
- フィルタ用の入力欄を追加する。
    - `TaskBoard` コンポーネント内で、ボードタイトルの下にフィルタエリアを追加する。
    - MUI の以下のコンポーネントを利用すること。
        - `TextField`（担当者名フィルタ）
        - `FormControl`, `InputLabel`, `Select`, `MenuItem`（優先度フィルタ）
        - レイアウト調整には `Box` や `Stack` を利用してよい。
    - フィルタ仕様
        - 担当者名フィルタ
            - 入力された文字列を `filterAssignee` に保持し、
                
                `task.assignee` に **部分一致** するタスクのみ表示する。
                
        - 優先度フィルタ
            - `filterPriority` が `'all'` の場合は全件表示。
            - それ以外の場合は、`task.priority === filterPriority` のタスクのみ表示する。
- 絞り込みロジックを実装する。
    - 既存の `tasks` 配列に対して、フィルタ条件を適用した配列 `filteredTasks` を作成すること。
        - 条件
            - `filterAssignee` が空でない場合
                
                → `task.assignee` に `filterAssignee` が含まれるタスクだけ残す。
                
            - `filterPriority` が `'all'` でない場合
                
                → `task.priority === filterPriority` のタスクだけ残す。
                
        - 両方の条件は **AND 条件** とする（両方を満たしたタスクのみ表示）。
    - 既存の `getTasksByStatus` 関数を修正し、
        
        `tasks` ではなく `filteredTasks` からステータスごとのタスクを取得するようにすること。
        
- `TaskColumn` 側の修正は不要。
    - `TaskBoard` で渡す `tasks` の中身がフィルタ済みに変わるだけでよい。

動作確認

- 担当者名フィルタに文字列を入力すると、その文字列を含む `assignee` を持つタスクだけが各列に表示されること。
- 優先度フィルタを「高 / 中 / 低」に切り替えると、対応する `priority` のタスクだけが表示されること。
- 担当者名フィルタと優先度フィルタを両方設定した場合、両方の条件を満たすタスクのみ表示されること。
- フィルタをクリアすると、全タスクが表示されること。

タスク管理ボードの上部に**新規タスク追加フォーム**を実装し、タイトル・担当者・優先度を入力してタスクを追加できるようにする。

    - `newTitle`: `string`
    - `newAssignee`: `string`
    - `newPriority`: `'low' | 'medium' | 'high'`
        - 初期値は `'medium'` など任意でよい。
- タスク追加フォームを実装する。
    - タイトルの下あたりに、以下の入力欄をまとめて配置する。
        - タスクタイトル入力用 `TextField`
        - 担当者入力用 `TextField`
        - 優先度選択用 `Select`（`low` / `medium` / `high`）
        - 「追加」ボタン（`Button`）
    - レイアウトには `Stack` や `Box` を利用してよい。
- 追加ボタン押下時の処理を実装する。
    - タイトルが空文字の場合は追加しない（そのまま return でもよい）。
    - 新しい `Task` オブジェクトを作成し、`setTasks` で `tasks` に追加する。

        - `status`: 初期は `'todo'`
        - `priority`: 入力された優先度
    - 追加後はフォームの入力値をクリアする。

動作確認

- タイトル・担当者・優先度を入力し、「追加」ボタンを押すと、Todo 列に新しいタスクカードが追加されること。
- 追加したタスクも、ドラッグ＆ドロップやフィルタ機能の対象として動作すること。

### ★応用（余裕がある人向け）:タスク削除機能の追加

タスク6まで実装できたら完了です。

ただ、もしまだ余裕があれば、各タスクカードに削除ボタンを追加し、タスクを削除できるようにしてみましょう。

作業内容

- `TaskBoard.tsx` に **タスク削除用のハンドラ関数**を追加する。
    - 例：`handleDeleteTask` 関数内で、指定された `id` のタスクを `setTasks` で除外する。
    - 物理的に削除するのではなく、`tasks.filter(task => task.id !== id)` のようにフィルタリングして削除を表現する。
- `TaskColumn` コンポーネントに `onDeleteTask` プロップを追加し、`TaskBoard` から渡せるようにする。
- 各タスクカード（`TaskCard` コンポーネントなど）に削除ボタンを表示する。
    - MUI の `IconButton` ＋ `Delete` アイコン、またはシンプルな「削除」ボタンを使ってよい。
    - ボタン押下時に `onDeleteTask(task.id)` を呼び出すことで、該当タスクを削除する。

動作確認

- 各タスクカードに削除ボタンが表示されていること。
- 削除ボタンを押すと、そのタスクカードが対応する列から消えること。
- 削除したタスクはフィルタやドラッグ＆ドロップの対象からも外れていること。

💡 **ヒント**

その1:処理の考え方

→削除機能を実装するためには、「どのコンポーネントがstateを持っているか」を意識する必要があります。

`tasks` の状態を持っているのは `TaskBoard.tsx` です。

そのため、「削除したい」というイベントも最終的には `TaskBoard` まで伝える必要があります。

これを踏まえて、イベントの流れを考えてみましょう。

その2:**useDraggable利用時の注意点（重要）**

→dnd-kitの `useDraggable` を**カード全体に適用している場合**、カード内に置いた削除ボタンのクリックイベントが、ドラッグ開始イベントと競合して正しく動作しないことがあります。

理由ですが、おそらくカード全体が「ドラッグ可能領域」になって、クリックしようとしてもドラッグが優先されてしまうのが原因です。

これはdnd-kitの仕様なので、React側のバグではありません。

この応用課題を実装する場合は、カード全体ではなく「ドラッグするための領域」と「ボタンをクリックするための領域」を分けて考える必要があります。

具体的に言うと、以下の通りです。

- カードの中に **「ドラッグ専用の小さなエリア（ハンドル）」** を用意して、
    
    その部分だけをドラッグ操作の対象にする。
    
- それ以外の部分（タイトルや削除ボタンなど）は、
    
    **通常のクリック操作だけが行われる領域**として扱う。
    

これらを踏まえて実装内容を良く見直し、修正してみてください。
