## Gitとは

これまで皆さんは、自宅のPCのローカル環境内でプロジェクトを作成し、プログラミングを行ってきました。

学習の中で、ファイルを提出するとなった場合も、対象のテキスト自体をコピーしたり、ファイルやプロジェクトを直接やり取りしていたのではないでしょうか。

このGitを利用することで、わざわざコピペしたり、ファイルを圧縮してLINEで送る…なんてことをしなくても、決められたコマンドを実行するだけで簡単にインターネット上に共有させることができるようになります。

Git単体ではインターネット上で管理ができないため、GitHubというWEBサービスも併用する必要がありますが、とにかくコマンドを覚えるだけで他のメンバーと同じソースコードを共有し合うことができる便利なツールであると覚えておいてください。

実務ではほぼ100%使うことになるし、新人エンジニアが「よくわからん」「なんか怖い」と忌み嫌う存在でもあります…

まずみなさんに覚えてほしいのは **慣れるまでは一つ一つのコマンドを確認にしてからEnterキーを押す** ことです。

なんなら「これで入力合ってますか？」と毎回確認してもらっても大丈夫です。

それくらいには慎重な操作する必要があると記憶に留めておいてください。

では、資料を見つつ進めていきましょう！

### Gitコマンドを体験

「GitBashの構成要素」からは少しだけ補足の座学を挟みます。

主に解説するのは以下のポイントです。

- お作法
    - `$` マークはコマンドであることを表すので入力しない
- Linuxコマンド
    - `cd` と `ls` コマンドについて
    - PowerShellだと `ls -a` の代わりに `ls -force` を利用する
- 初期設定
    - 必ず新規プロジェクトを作成してから行うように
    - `init` を実行する場所を間違えないように注意喚起

以降は全体の進捗を見つつ、全員が問題なく進められるように資料に沿って進行していきましょう。

### チームでやってみる

ここも基本的には資料に沿って進めてください。

ポイントとしては…

- 一人を指名してGitHub上のプロジェクトを共有してもらう
    - 一番進捗が良い人
    - 他メンバーにメールアドレスを共有してもらいプロジェクトへ招待してもらう
- プルリクエストのマージはローテーションにする
    - 出した人とは別の人に確認してもらうようにしましょう

【流れについて】

1. 一人を指名しプロジェクトを共有してもらう
    1. 他メンバーにSlackにてメールアドレスを送ってもらう
2. 招待を確認しcloneしてもらう
    1. この時、同名のプロジェクトが存在するため別のフォルダを指定させる
    ※Git管理下の中でcloneすると面倒なので注意してください
    2. clone後に `cd` で移動、 `npm install` を忘れずに！
3. プロジェクト共有者はローカル上で `develop` ブランチを作成しリモートにプッシュ
4. メンバーは `develop` ブランチをローカルに持ってくる
5. 全メンバーが `develop_YOURNAME` ブランチを作成し `App.tsx` を各自修正する
6. 各自のブランチをリモートにプッシュさせ、順番にプルリクエスト⇒マージ
7. コンフリクトが発生するので次の手順へ

### Github Desktopの使い方

Gitの仕組みを理解したところでそれらをGUIで出来るツールとして紹介する。

社内の案件でも使うので、今後の操作はこれを使用してもらう前提で進める。

画面を見せながら以下の流れでサクサク進めてGUIの利便性を伝えるようにしてください。

また、サクサク操作は出来るが**コミット内容は必ず確認すること**を念押ししてください。

1. Github Desktopをインストール
https://docs.github.com/ja/desktop/installing-and-authenticating-to-github-desktop/installing-github-desktop
2. Githubアカウントの連携（GithubDesktopの操作）
3. デモ操作用プロジェクトを作る（npm create vite@latest）
4. プロジェクトをGit管理する（GithubDesktopの操作）
「Create a New Repository on your Hard Drive…」から追加する
5. Github上にリモートリポジトリを作成（GithubDesktopの操作）
Ctrl+P でリモートリポジトリの作成＆push
6. ブランチを切り替える（GithubDesktopの操作）
Ctrl+Shift+Nで「sampleBranch」ブランチを切り、チェックアウト
7. App.tsxを適当に編集（VSCodeの操作）
8. コミットメッセージを編集しコミットする（GithubDesktopの操作）
Ctrl+Enterでコミット
9. リモートリポジトリにpush（GithubDesktopの操作）
Ctrl+Pでpush
10. プルリクの作成（Githubの操作）
Ctrl+RでGithubが立ち上がるので、プルリクを作成
作成したプルリクを承認してマージ
11. ブランチをmainに切り替える（GithubDesktopの操作）
Ctrl+Bで「main」ブランチに切り替える
12. リモートの変更を取り込む（GithubDesktopの操作）
Ctrl+Shift+Tでfetch
Ctrl+Shift+Pでpull（merge）

### コンフリクト解消の手順（時間があれば）

1. 2つブランチを切りどちらか一方にチェックアウト（GithubDesktopの操作）
task01で修正してPRまで
task02で修正してPRまで（コンフリクトが起きる）
2. コンフリクトを解消する（GithubDesktopの操作）
Ctrl+Shift+Uでmasterブランチを取り込む（コンフリクト発生）
VSCodeでコンフリクトを解消
3. pushまでする（GithubDesktopの操作）
Ctrl+Enterでコミット
Ctrl+Pでpush
4. PRを取り込む（Githubの操作）
