## 概要

WEB APIを通じて行ったCRUD処理の正体を知ってもらうためにPostgreSQLを使ったデータ操作を行う。

レベル感としては、CRUDのこれはこの構文を使うよ、動かしてみよう、ほんとにできたね！という塩梅で全体的にやんわりさせる方針となる。

## データベースとは

### “データ”を保存しておくための仕組み

今回のカリキュラムでは、「データベース」を触ってもらいます。
前回体験した「CRUD処理」の正体とも言える大切な内容となります。

実際に手元で動かしつつ、データ操作を身に着けていきましょう！

データベースはその名の通りで、データを保存しておくための仕組みです。
Reactを利用して通信し、取得や登録を行ってもらいましたが、基となるデータは全てデータベースが管理していたのです。

今回は、PostgreSQLというデータベースを利用して各種操作を体験してみてください。

### 絶対に覚えてほしいこと

これから資料を見つつ実際にデータベースを操作してもらいますが、次のキーワードは絶対に覚えてください！

- SQL、クエリ文
    - データベースを操作するための命令文
- INSERT
- SELECT
- UPDATE
- DELETE

このカリキュラム以降にデータベースを直接クエリ文を使って操作することはありません。
ですが、Reactを触っていく中で、各種命令文は目にすることになるため、意味合いをしっかりと記憶しておかないと後々自分が苦しむことになります。

そのためにも、絶対に覚えてください。
もう一度言います。

- INSERT
- SELECT
- UPDATE
- DELETE

この後自分で操作をしながら、意味合いとキーワードを確実に頭に叩き込んでいきましょう！

⇒資料を見ながら進めてもらう

### 確認テスト

- データベースとはなにか？
    - WEBサービスなどのデータを管理する仕組み
    - データの登録や取得、更新などの操作が可能
- CRUDとはなにか？
    - データベースでよく利用する操作の略称
        - 4つの操作の頭文字を取ってる
    - WEBシステムの根幹を担っている
- Create（新規登録）に利用するクエリ文は？
    - INSERT
- Read（取得）に利用するクエリ文は？
    - SELECT
- Update（更新）に利用するクエリ文は？
    - UPDATE
- Delete（削除）に利用するクエリ文は？
    - DELETE
- 論理削除って？
    - データを”消したこと”にする削除方法
    - 専用の管理フラグデータを用いる
    - 基本的には `deleted` という名称
    - `false` と `true` で管理している
- 物理削除って？
    - DELETE文を利用した完全に削除する方法
