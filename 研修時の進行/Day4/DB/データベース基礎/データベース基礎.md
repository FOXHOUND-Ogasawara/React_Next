## 操作してみよう

### まずはじめに

データベースでは、データ操作を行う際は、以下の4つの命令文を利用することが多いです。

- 挿入： `INSERT`
    - データを新規登録する命令
- 選択： `SELECT`
    - データを取得する命令
- 更新： `UPDATE`
    - データを更新する命令
- 削除： `DELETE`
    - データを削除する命令

ここまで学習した皆さんなら、なんとなく見覚えがあるのではないでしょうか？

Reactを使ってデータの登録や取得、更新や削除を行ってもらいましたが、これらの操作を「CRUD処理」と呼びます。

CRUDは、以下の頭文字で構成されています。

- C： `Create`
- R： `Read`
- U： `Update`
- D： `Delete`

CreateとReadについては、名前が異なりますが、やりたいこと自体は新規作成やデータ取得なので意味合いは同じですよね。

実は、みなさんに体験してもらったデータを操作するCRUD処理というのは、データベース上のデータを操作するための仕組みなのです。

AmazonやYouTubeのように、インターネット上に公開されているサービスの多くは「データ」がたくさん存在しているのは容易に想像ができるはずです。

この「データ」は、データベースというツールが記憶していて、ユーザーはそのデータベースから取得されたデータを閲覧したり、データベースに自身の情報を登録したりしているのです。

前回の学習では、そんなデータベースを意識せずにWEB APIを利用して操作していましたが、今回はデータを管理している本体であるデータベース自体を実際に操作してみましょう。

### データを登録する

データ操作を行うためには、データベースを操作するために命令文を入力し、実行する必要があります。

データを登録する命令文を実行するためにも、まずはエディタを開いてみましょう。

pgAdminの左上に `Tools` というタブがあるので選択し、その中に `Query Tool` という項目があるのでクリックしてください。

![image.png](image%2036.png)

クリックすると、空白のページが表示されるので、そこに以下の命令文を記述し実行を行ってみましょう。

```sql
INSERT INTO
  users (name, age)
VALUES
  ('yamada', 22)
;
```

![image.png](image%2037.png)

実行ボタンを押すことで、記述してもらった命令文が実行されるのですが、成功したかどうかは下部のメッセージ欄に表示されます。

英語ではありますが、成功とでれば完了です。

![image.png](image%2038.png)

無事データの登録が完了したら、別のデータも登録してみましょう。

```sql
INSERT INTO
  users (name, age)
VALUES
  ('tanaka', 30),
  ('sato', 33)
;
```

先ほど記述したINSERT文の内容を削除するか、 `ctrl` + `/` キーを押してコメントアウトしておいてください。

ワークスペース上に複数の命令文があると、すべて実行されてしまうので、もう一人のyamadaさんが重複登録されてしまうので気を付けましょう。

### データを取得する

次の命令文をコピーして実行してみてください。

※INSERT文は削除するか、コメントアウトするのを忘れずに

```sql
SELECT
  name,
  age
FROM
  users
;
```

実行するとテーブル形式で先ほど登録したデータが一覧表示されます。

WEB APIを使ってデータを取得する際も、実はサーバー側でこのSELECT文が実行されていたりします。

また、データベースへの命令文を SQL文 と呼ぶので覚えておきましょう。
ちなみに、クエリとも呼ばれたりするのでこちらも合わせて覚えておきましょう。

一度WEB APIを通じて、特定のIDだけで絞りこんだり、並び順を変えたりしましたが、こういった少し特殊なデータ取得も可能です。

次のクエリを確認してみてください。

```sql
SELECT
  *
FROM
  users
WHERE
  age >= 30
;
```

先ほどのSELECT文よりも若干複雑な印象を受けたかもしれません。

まず、冒頭に登場する `*` ですが、これは「対象のテーブルにあるカラムを全て指定する」というキーワードです。

テーブルというのは最初に作った `users` のことですね。

カラムは `name` や `age` といった、何のデータなのかという部分です。

逆に考えると、 `SELECT name FROM users` というクエリを実行するとテーブルに登録されている情報から名前だけを取り出すことができるわけです。

次に、 `WHERE` というキーワードですが、これが絞り込む条件を加えるための文法です。
Reactで使っていた `if` に近いイメージですね。

`age >= 30` と設定することにより、年齢が30以上のデータを対象として絞り込むという条件がプラスされるわけです。

数値を変えたりして実際に試してみてください。ちなみに、 `>=` 以外にもいくつかあるので紹介しておきます。

| 記号 | 意味 |
| --- | --- |
| `>` | より大きい |
| `<` | より小さい |
| `>=`  | 以上 |
| `<=` | 以下 |
| `=` | 同じかどうか |

これらに加えて、「かつ」や「または」という条件を加えることも可能です。

```sql
SELECT
  *
FROM
  users
WHERE
  name = 'yamada'
  AND age >= 30
;
```

こういった条件を組み合させることで、特定のデータだけを取得することが可能になります。

### データを更新する

データを更新するために次のようなクエリを記述します。

```sql
UPDATE
  users
SET
  age = 21
WHERE
  id = 3
;
```

すでになんとなく読み解けるのではないでしょうか？

データを更新するということは、更新する対象を指定する必要があるため、 `WHERE` での条件設定を忘れないように注意しなくてはなりません。

もし、仮に忘れてしまうと…対象としたテーブル内の全てのデータに更新が入ってしまうという恐ろしい恐怖体験を目にすることになります…

SELECT文と比べると、扱っているキーワードが異なりますが、意味合いとしては英語の通りです。

`UPDATE` から始まり、どのテーブルを対象とするのか？ `SET` でどのカラムをどんな値で上書きするのか？ `WHERE` でテーブルの中のどのデータに対して実行したいのか、これらを設定しているわけです。

### データを削除する

データを削除する場合には注意が必要です。

まずは以下のクエリを確認してみてください。

```sql
DELETE
FROM
  users
WHERE
  id = 5
;
```

形式としては、先ほどの `UPDATE` に似ていますが、カラムの指定がありません。

また、削除する対象を `WHERE` を使って絞り込んでいますね。削除する際も更新と同様に「どのデータが対象なのか」を明確に指定する必要があります。

もし、対象を指定しないで削除を実行すると…

なんとなくわかりますよね。では、さらに次のクエリも見てください。

```sql
UPDATE
  users
SET
  deleted = true
WHERE
  id = 5
;
```

どこかで見覚えがないでしょうか？

WEB APIを使って、ユーザー情報を削除した際に「削除したことにする」というデータ操作をしたのを覚えていますか？

データベースにおいても、データを削除する際は `DELETE` を使うのではなく、 `UPDATE` を使うケースが圧倒的に多いです。

実際に削除してもらうとわかるのですが、削除した場合は **簡単にデータを戻すことができません**。バックアップなどを取っていない場合は**完全に消滅します**。

完全に消えてしまうと色々と後で面倒なことになるため、基本的には削除は専用のカラムを更新して「消したことにする」のが一般的であると覚えておきましょう。
