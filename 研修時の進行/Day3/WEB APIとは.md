## WEB APIとは

そもそもWEB APIとはなんでしょうか？答えられる人はいますか？

WEBエンジニアの界隈ではWEBを付けずにAPIと呼ぶこともあるのですが、簡単に言ってしまうとインターネット上に公開されている便利なメソッドのようなものです。

いくつかのルールはありますが、URLにアクセスすることで簡単にデータを取得したりできるため、よく利用されます。
Reactコースの学習内容にも、fetchを利用してURLにアクセスしたことがあるかと思いますが、あれもWEB APIを叩いてデータを取得していたわけですね。

今回の座学では、世の中にたくさん存在しているWEB APIをどうやって使うのか？という部分について学習していきます。

WEBエンジニアとして仕事をしてもらうとなると、このWEB APIの存在は必ず登場します。
その時に備えてしっかりと身に付けていきましょう。

ちなみに、WEB APIは人が作ったものなので、逆に考えると皆さんも作ることが可能です。
もしかすると実務ではAPIを使うのではなく、作る立場になる可能性も全然あるので、そこも覚えておきましょう。

### HTTPメソッドのおさらい

まずは、リクエストの種類について改めておさらいします。

リクエストには、`GET` `POST` という種類が存在しています。

実はこれ以外にも `PUT` `DELETE` なども用意されていて、リクエストを投げる際に「なにをするのか」に応じて使い分けることが多い。

簡単に各役割を覚えておこう。

GET：一般的なリクエストで、基本的にはGETを使う、データを取得する際に利用される

POST：新規登録や更新時に利用される

PUT：データの更新時に利用される

DELETE：データを削除するときに利用される

今回紹介したWEB APIを提供しているサイトでは、GETを使うことになるので何個か種類があるんだなというのは覚えておきましょう。

### リクエストの形式

リクエストとレスポンスというキーワードは覚えてますか？

クライアントからサーバーへ要求することをリクエストと呼び

サーバーからクライアントへ応答を返すことをレスポンスと呼びます

まずはリクエスト…クライアントからサーバーに対してデータや指示を送信する仕組みであるリクエストの構成要素を見て行きましょう。

- URL
    - どのリソースに対してリクエストを行うかを示す
- HTTPメソッド
    - どの操作を行うかを示す
- Header
    - APIキーやコンテンツの形式（JSONなど）を指定する
- Body
    - POSTやPUTで送る場合にデータを渡す部分（ユーザーデータなど）

```tsx
GETの例
https://api.example.com/weather?q=Tokyo&appid=API_KEY
```

GETでリクエストする場合は、URLの後ろに「?」を付けることで直接データを含めた形式にすることが可能です。

「?Key=Value」の形式のURLに含める文字をクエリパラメータと呼んだりするので覚えておきましょう。

### レスポンスの形式

次はレスポンスについても見て行きましょう。

レスポンスは、サーバーがクライアントに返すデータのことです。構成要素としてはこんな感じです。

- ステータスコード
    - リクエストが成功したり失敗したかを表す（200や500など）
- Header
    - レスポンスのメタ情報（JSONなどの形式）
- Body
    - データ（JSON形式）

```tsx
{
	"weather": {
		"description": "clear sky",
		"temperature": 25.3
	}
}
```

上記の例では、天気の情報がレスポンスとして返され、JSON形式で「晴れ」の天気と温度が取得できていることを表しています。

### ドキュメントを確認してみる

なんとなく必要なキーワードと形式を知ったので、改めてドキュメントを覗いてみましょう。

ひとまずはWeather APIのドキュメントを確認してみてください。

- どのようにリクエストする必要があるのか
- どのようなレスポンスが返ってくるのか

これらを読み解くことができるようになったのではないでしょうか？

URLの情報、HTTPメソッド、必要となるHeader情報…これらがどこに書いてあるのかを見つけて実際にデータが取得できるかを確かめてみましょう。
