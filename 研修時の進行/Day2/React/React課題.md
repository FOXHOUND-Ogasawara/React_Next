# React課題

## 概要

初日に実施することを想定したReactの復習用の課題です。
ベースとしてはReactコース内のExamと能力確認用Examとなっており、研修カリキュラムの内容が布石として体験できるお題となっています。

### 目安時間

タスク4までで5時間程度あれば実装できるボリューム感にしています。

目安時間としては
　環境構築とタスク確認+言語化：60分
　商品データの表示機能（タスク1）：70分
　カテゴリー絞り込み機能（タスク2）：50分
　カート機能の実装（タスク3）：90分
　ローディング機能（タスク4）：30分

### 本課題の目的

- 自己学習ではなく、仕事であることを認識させる
    - 時間制限を短く設けることで以下を意識させる
        - Reactの知識が身についていないとヤバい
        - 自分のペースで進めるのはマズい
    - 学ぶ < 終わらせる
- 質問や報告の練習
- 現時点のスキル確認
    - 技術力
        - Reactのベース知識が身についているか
        - PCの操作に問題はないか
    - 調査力
        - AIの利用はできるか
        - 必要な情報を自発的に探せるか
        - 時間を無駄にせず効率的な調査が行えるか
    - コミュニケーション能力
        - 報連相が適切に行えるか

## 全体の流れ

1. 業務指示としての内容を伝える
    1. 展開資料「研修を受けるにあたって」の内容
2. 展開資料「React課題（資料）」を展開
    1. 概要説明
    2. 期限の設定
3. 環境構築
    1. Reactが動作することを確認
    2. 本課題用のプロジェクト作成
    3. プロジェクトの動作確認（画面が出るかを確認）
4. タスク確認
    1. やるべきことをリスト化（30分くらい時間取ってもよい）
        1. 提出してもらう
    2. 実装に着手
        1. 各タスク完了ごとに報告
        2. 不明点があれば質問

上記の流れで指示をお願いします。

初日は日報の説明を実施するため 17:45 までの作業となります。
また、終礼時に資料の最下部にある「明日に向けて」の調査を宿題として課してください。

本課題については、タスク3まではマストし、終わらない場合は延長ではなく「自己学習の一環」として実装は行うように指示してください。

◆業後の作業について◆

- 原則残業は禁止です
- 自宅でのタスク着手も禁止です

タスクが終わらずどうしても残作業が発生する場合は、期限とどこまでをゴールとするかを講師と相談してください。そのうえで残って進める場合は「自己学習」となります。
必ず退勤し、講師や上長の許可を取り、何時まで残るのかを明確に報告させてください。

### 実施するにあたって

目的にもあるように、本課題はあくまでも現時点のスキル確認および仕事としてやらなければいけない意識付けとしての立ち位置となります。

お題を展開する前に、以下を業務指示として伝えてください。

- **自身がこれから何をするべきなのかを把握し言語化を行う
そのうえで報告したり、不明点を質問すること**
    - フォーマットについては別途資料を参照
- **振られたタスクを終わらせるための動きを徹底すること**
    - 実務では納期があり、複数人で開発を行うことになる
    - 自分の遅れは全体の遅れに繋がり、周囲やお客様に迷惑をかける（売り上げに影響）
    - 自分でやらなきゃという考えは捨て、終わらせるための行動を取ってください
    ※残業しろではなく、時間内に終わる工夫をしろという意味
- **報連相を徹底すること**
    - みなさんが思っている10倍は求められます
    - 質や粒度は研修内で身に着ければよいのでまずは報告や質問、相談をしてください
    - 不明点の質問や進捗遅れの相談以外にも、順調なのかも報告するようにしてください
