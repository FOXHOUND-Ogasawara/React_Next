# WEBの基礎【資料】

### **資料の概要**

1. **クライアントとサーバーの定義**
    - クライアント: 利用者側のデバイスやアプリケーション。
    - サーバー: データを保存し、クライアントからのリクエストに応じてデータを提供するコンピュータ。
2. **リクエストとレスポンスの流れ**
    - リクエスト: クライアントがデータを要求するプロセス。
    - レスポンス: サーバーがクライアントにデータを返すプロセス。
3. **クライアントとサーバーのやり取りの例**
    - 検索の例（Google検索、Amazonの商品検索）
    - サーバーがどのように情報を返すかを説明する図解。

### **資料に含める項目**

- クライアントとサーバーの関係図
- リクエストとレスポンスのフロー図
- 各プロセスの簡単な説明

## クライアントとサーバー

### クライアントの役割

クライアントとは…

ユーザーが直接利用する端末やアプリケーションのこと。
サーバーに対してリクエストを送信します。
すごく簡単に言えば、皆さんが使ってるPCやスマートフォンを指します。

- リクエストの送信
    
    クライアントは、必要な情報やサービスをサーバーにリクエストします。
    たとえば、WEBブラウザがWEBページを表示するためにサーバーにデータを要求するような動きのことです。
    
- ユーザーインターフェースの提供
    
    サーバーから取得したデータを利用者に見やすく、わかりやすく表示して、操作を簡単にする役割も持っています。
    

### サーバーの役割

サーバーとは…

クライアントからのリクエストに応答し、情報やデータを提供する役割を担います。
すごく簡単に言えば、どこかにある管理用PCのようなものです。

- リクエストに応答
    
    クライアントから送られてくるリクエストを受け取り、必要な情報を返します。
    
- データの管理と提供
    
    大規模なデータを安全に管理し、クライアントからのリクエストに応じてデータを提供します。
    

### クライアントとサーバーのやり取り

クライアントとサーバーはHTTP通信を用いてやり取りされます。

- 通信のサイクル
    
    PCやスマホのようなクライアントからサーバーに対してリクエストを送信
    サーバー側で処理が行われた後にクライアントにレスポンスを返す

## リクエストとレスポンス

WEB通信において、クライアントからサーバーに対して情報を要求することを「リクエスト」と呼び、サーバーがその要求に応じて返すデータを「レスポンス」と呼びます。

WEBサービスにとって、非常に重要なキーワードであり、WEBを支える大切な仕組みです。
何度も登場するキーワードなので絶対に覚えておいてください。

### リクエスト

リクエストには様々な形式が用意されています。

- `GET`：データの取得
- `POST`：データの作成
- `PUT`：データの更新
- `DELETE`：データの削除

<aside>
💡

これらを「HTTPメソッド」と呼びます。

上記のほかにも種類があり、データの一部を更新する PATCH などがある。

</aside>

リクエストは以下の構成要素によって成り立っています。

- URL： `http(s)://` から始まるよくみる通信の宛先
- ヘッダー：認証情報やデータの形式などを指定する
- ボディ：POSTやPUTなどで送信される実際のデータ

```json
POST /api/v1/users HTTP/1.1
Host: example.com

// ヘッダー
Content-Type: application/json
Authorization: Bearer YOUR_API_KEY

// ボディ
{
	"name": "John Doe",
	"age": 30
}
```

### データの送り方に違いがある

HTTPメソッドには、複数の種類が用意されており、どんなことをしたいのか？に応じて使い分けるのが一般的です。

その中でも、 `GET` とそれ以外ではデータの送り方に大きな違いがあるので合わせて覚えておきましょう！

- `GET`：URLの一部としてデータを送信する
    - URL上に直接データが入るため、ボディが不要でリンクとして共有できる
    （例：AmazonやYouTubeの商品や動画など）
    - URLを見れば誰でも送りたい情報を見ることができる
- それ以外：リクエストボディとしてデータを送信する
    - URLだけでは何を送るのか見ることができない

URLに直接データを含めるのか、リクエストボディに含めるのかという違いがあります。

クレジットカードなどの他の人に見られたくない情報などをURLに含めてしまうと危険であることは簡単に想像が付くと思いますが、データの登録や更新と言った操作はPOSTなどを利用するのが一般的です。

### レスポンス

レスポンスには、リクエストが成功したのかどうかや、処理した結果などの情報が含まれています。

- ステータスコード
    - `200 OK`：成功
    - `3xx`：リダイレクト系
    - `404 Not Found`：ページが存在しない
    - `500 Internal Server Error`：サーバー側でエラー発生
- ヘッダー：レスポンスの形式などが含まれる
- ボディ：サーバーが返すデータ

```json
HTTP/1.1 200 OK

// ヘッダー
Content-Type: application/json

// ボディ
{
	"id": 1,
	"name": "John Doe",
	"age": 30
}
```
